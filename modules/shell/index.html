    Shell e Execucao¶

    Modulos de execucao de comandos e gerenciamento do ciclo de vida do agent.

    Resumo¶

    Modulo Plataformas Descricao
    shell Windows, Linux, macOS Sessao shell interativa
    shell.exec Windows, Linux, macOS Execucao de comando unico
    shell.selfdestruct Windows, Linux, macOS Auto-remocao completa do agent

    shell¶

    Nome: shell Plataformas: Windows, Linux, macOS Descricao: Inicia uma sessao shell interativa no alvo, usando o shell padrao da plataforma ou um binario customizado da whitelist.

    Argumentos¶

    Nome Tipo Obrigatorio Default Descricao
    shell string opcional /bin/sh (Unix) / cmd.exe (Windows) Binario shell a utilizar
    input string opcional - Dados a enviar para stdin do shell

    Shells permitidos (whitelist):

    • Windows: cmd.exe, powershell.exe, pwsh.exe
    • Unix: /bin/sh, /bin/bash, /bin/zsh, /usr/bin/bash, /usr/bin/zsh

    Se o shell solicitado nao estiver na whitelist, o default da plataforma e usado com um log de warning.

    Retorno¶

    {
  "output": "stdout + stderr combinados",
  "data": {
    "status": "success | error | cancelled"
  }
}

    Exemplo de uso¶

    # Shell padrao
module shell

# PowerShell
module shell --shell powershell.exe --input "Get-Process | Select-Object -First 5"

# Bash com comando
module shell --shell /bin/bash --input "id && whoami && hostname"

    Notas¶

    • Sessoes sao exclusivas por agent: apenas um operador pode ter uma sessao ativa por vez (SessionManager)
    • O contexto pode cancelar a sessao -- nesse caso status retorna cancelled
    • WaitDelay de 3 segundos forca o fechamento dos pipes apos kill do processo
    • Stdin e fechado apos enviar input para sinalizar EOF

    shell.exec¶

    Nome: shell.exec Plataformas: Windows, Linux, macOS Descricao: Executa um unico comando no shell da plataforma e retorna o resultado.

    Argumentos¶

    Nome Tipo Obrigatorio Default Descricao
    command string obrigatorio - Comando a executar
    timeout string (segundos) opcional 30 Timeout de execucao

    O comando e executado via cmd.exe /c (Windows) ou /bin/sh -c (Unix).

    Retorno¶

    {
  "output": "stdout + stderr combinados",
  "data": {
    "status": "success | error | timeout",
    "exit_code": "0",
    "error": "mensagem (somente quando status=error)"
  }
}

    Exemplo de uso¶

    # Comando simples
module shell.exec --command "whoami /all"

# Com timeout customizado
module shell.exec --command "ping -c 100 10.0.0.1" --timeout 120

# Listagem de diretorio
module shell.exec --command "ls -la /tmp"

    Notas¶

    • Diferente do shell (interativo), shell.exec executa um comando e encerra
    • Se o timeout for atingido, status retorna timeout e o processo e terminado
    • WaitDelay de 3 segundos garante fechamento limpo dos pipes

    shell.selfdestruct¶

    Nome: shell.selfdestruct Plataformas: Windows, Linux, macOS Descricao: Remove completamente o agent do alvo, incluindo persistencia, artefatos, e o proprio binario.

    Argumentos¶

    Nome Tipo Obrigatorio Default Descricao
    self_path string opcional auto-detectado via os.Executable() Caminho do binario do implant

    Retorno¶

    {
  "output": "self-destruct completed 3 steps, 0 errors",
  "data": {
    "status": "terminated",
    "steps": "cleanup:...;overwrite:done;remove:done",
    "step_count": "3",
    "error_count": "0",
    "errors": "(somente quando error_count > 0)"
  }
}

    Sequencia de Execucao¶

    1. Cleanup -- executa persist.cleanup para remover toda persistencia e artefatos rastreados
    2. Overwrite -- sobrescreve o binario do implant com dados aleatorios (crypto/rand) em chunks de 32KB
    3. Remove -- deleta o binario do disco (implementacao platform-specific)
    4. Return -- envia resultado final para o servidor
    5. Exit -- agenda terminacao do processo via goroutine (para que o resultado seja enviado primeiro)

    Exemplo de uso¶

    # Auto-destruct padrao (detecta binario automaticamente)
module shell.selfdestruct

# Com caminho explicito
module shell.selfdestruct --self_path "C:\Windows\Temp\svc.exe"

    Notas¶

    • Operacao irreversivel -- o agent sera permanentemente removido
    • O overwrite com dados aleatorios impede recuperacao forense do binario original
    • A remocao do binario e platform-specific:
      • Windows: usa tecnicas de auto-delecao (MoveFileEx, etc.)
      • Unix: os.Remove() direto
    • O processo termina com exit(0) apos o resultado ser enviado ao servidor
    • Se o cleanup falhar parcialmente, os erros sao reportados mas a sequencia continua
