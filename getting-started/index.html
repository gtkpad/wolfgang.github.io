    Primeiros Passos¶

    Wolfgang e um framework C2 (Command and Control) para testes de penetracao autorizados, escrito em Go. Ele fornece a infraestrutura completa para operacoes ofensivas: gerenciamento de implants, comunicacao criptografada, execucao de modulos e exfiltracao de dados.

    Uso Legal

    Wolfgang foi projetado exclusivamente para testes de penetracao autorizados e pesquisa em seguranca. O uso deste software contra sistemas sem autorizacao explicita e ilegal. Sempre obtenha permissao por escrito antes de qualquer operacao.

    Visao Geral da Arquitetura¶

    O framework e composto por tres binarios que trabalham em conjunto:

    Componente Binario Funcao
    Teamserver wolfgang-server API REST, gerencia listeners, armazena dados em SQLite, compila implants
    Operator CLI wolfgang-client REPL interativo, conecta ao teamserver via HTTPS + WebSocket
    Implant wolfgang-implant Agent deployado no alvo, check-in periodico, execucao de modulos

    Fluxo de Operacao¶

    1. O operador se autentica no teamserver via wolfgang-client
    2. O teamserver gerencia listeners que aguardam conexoes dos implants
    3. O implant faz check-in periodico no listener, recebendo tasks pendentes
    4. Tasks sao executadas como modulos no alvo (shell, recon, exfil, etc.)
    5. Resultados retornam criptografados ao teamserver
    6. O operador visualiza resultados em tempo real via WebSocket

    Pre-requisitos¶

    Software¶

    Requisito Versao Minima Notas
    Go 1.25+ Necessario para compilar todos os binarios
    make qualquer Build system
    garble (opcional) Ofuscacao de strings no implant
    Git qualquer Clone do repositorio

    Plataformas Suportadas¶

    Teamserver e Client podem rodar em qualquer plataforma suportada pelo Go:

    • Linux (amd64, arm64)
    • macOS (amd64, arm64)
    • Windows (amd64)

    Implants podem ser cross-compilados para:

    OS Arquiteturas Formato
    Linux amd64, arm64, 386 ELF
    Windows amd64, arm64, 386 PE (.exe)
    macOS amd64, arm64 Mach-O

    Rede¶

    • O teamserver expoe a API na porta 8443 (configuravel)
    • Listeners usam portas separadas (ex: HTTPS na 443, DNS na 53)
    • O client precisa de acesso HTTPS ao teamserver
    • Implants precisam de acesso de saida ao listener correspondente

    Conceitos Essenciais¶

    Antes de comecar, familiarize-se com a terminologia usada no Wolfgang:

    Termo Descricao
    Teamserver Servidor central que coordena toda a operacao
    Operator Usuario autenticado que interage com o teamserver
    Listener Endpoint que recebe conexoes dos implants (HTTPS, DNS, WebSocket)
    Implant Binario deployado no alvo que executa tasks
    Agent Instancia de um implant registrado no servidor
    Task Unidade de trabalho enviada a um agent (ex: executar comando, exfiltrar arquivo)
    Module Codigo que implementa uma capacidade especifica (shell, recon, evasion, etc.)
    Transport Protocolo de comunicacao entre implant e listener
    Check-in Conexao periodica do implant ao listener para buscar tasks pendentes
    TOFU Trust On First Use -- modelo de confianca para certificados TLS
