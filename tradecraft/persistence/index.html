    Persistência¶

    Persistência garante que o implant sobreviva a reboots, logoffs e atualizações do sistema. Sem persistência, o operador perde o acesso assim que o processo do implant é encerrado -- todo o esforço de acesso inicial precisa ser repetido.

    O Wolfgang implementa mecanismos nativos de cada sistema operacional, utilizando funcionalidades legítimas de auto-start em vez de técnicas exóticas. A escolha deliberada de mecanismos "barulhentos mas confiáveis" reflete a realidade de que persistência precisa funcionar 100% das vezes -- uma técnica sofisticada que falha em 5% dos reboots compromete a operação.

    Matriz de Plataformas¶

    Técnica Windows Linux macOS Privilégio MITRE
    Registry Run Keys User/Admin T1547.001
    Scheduled Tasks Admin T1053.005
    Cron Jobs User/Root T1053.003
    LaunchAgent/Daemon User/Root T1543.001

    Cada técnica tem um perfil diferente de visibilidade e confiabilidade:

    Windows¶

    • Registry Run Keys: Simples e confiável. Detectável por qualquer ferramenta de autoruns, mas funciona em 100% dos cenários sem necessidade de privilégio elevado (HKCU).
    • Scheduled Tasks: Mais flexível (triggers por horário, logon, evento). Requer admin para triggers de sistema. Gera logs (Event ID 4698).

    Linux¶

    • Cron: Universal em distribuições Linux. Dois modos: user crontab (sem root) e system crontab (/etc/cron.d/, com root). Simples de auditar com crontab -l.

    macOS¶

    • LaunchAgents: Equivalente ao cron, mas integrado ao launchd. LaunchAgent (user-level) vs LaunchDaemon (root-level). Plist format requer atenção ao Label e ProgramArguments.

    Abordagem Operacional¶

    Quando usar persistência¶

    Não instale persistência automaticamente. Avalie o contexto:

    • Acesso inicial instável: Persistência imediata para não perder o foothold
    • Ambiente monitorado: Considerar se o risco de detecção vale o benefício
    • Objetivo de curta duração: Pode ser desnecessário se a operação é rápida
    • Múltiplos implants: Persistência em um, os outros são descartáveis

    Cleanup¶

    Toda persistência deve ser removível. O operador é responsável por remover artefatos de persistência ao final da operação para limitar a exposição.
