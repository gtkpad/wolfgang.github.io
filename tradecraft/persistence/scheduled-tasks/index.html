    Scheduled Tasks¶

    O que é¶

    O Task Scheduler do Windows permite agendar a execução de programas com base em triggers (horário, logon, evento do sistema, idle, etc.). É o mecanismo de auto-start mais flexível do Windows, usado por atualizações do sistema, backups, e manutenção automatizada.

    Para persistência, scheduled tasks oferecem vantagens sobre Run keys: suportam múltiplos triggers, podem rodar como SYSTEM (sem sessão de usuário), e sobrevivem a logoffs (não apenas reboots).

    Por que funciona¶

    O Task Scheduler é um serviço de sistema (Schedule) que roda como SYSTEM e processa tarefas definidas em XML. As tarefas são armazenadas em:

    • C:\Windows\System32\Tasks\ (arquivos XML)
    • HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule\TaskCache\ (metadados)

    Quando um trigger é ativado, o serviço executa a ação definida na tarefa. O processo filho herda o security context configurado (conta do usuário ou SYSTEM).

    Há duas formas de criar tarefas:

    1. schtasks.exe: Command-line tool. Mais simples, mas gera eventos de processo.
    2. COM API (ITaskService): Criação programática sem spawnar processos filhos.

    O Wolfgang usa schtasks.exe pela simplicidade e compatibilidade universal.

    Implementação no Wolfgang¶

    O módulo (pkg/module/persist/schtask_windows.go) cria tarefas via schtasks.exe com três tipos de trigger:

    func platformCreateSchTask(name, exePath, trigger string) error {
    var scheduleArgs []string

    switch trigger {
    case "onlogon":
        scheduleArgs = []string{
            "/Create", "/F",
            "/TN", name,
            "/TR", exePath,
            "/SC", "ONLOGON",
            "/RL", "HIGHEST",
        }
    case "daily":
        scheduleArgs = []string{
            "/Create", "/F",
            "/TN", name,
            "/TR", exePath,
            "/SC", "DAILY",
            "/ST", "09:00",
            "/RL", "HIGHEST",
        }
    case "hourly":
        scheduleArgs = []string{
            "/Create", "/F",
            "/TN", name,
            "/TR", exePath,
            "/SC", "HOURLY",
            "/RL", "HIGHEST",
        }
    default:
        return fmt.Errorf("unsupported trigger: %s", trigger)
    }

    cmd := exec.Command("schtasks.exe", scheduleArgs...)
    output, err := cmd.CombinedOutput()
    if err != nil {
        return fmt.Errorf("schtasks create failed: %s: %w", string(output), err)
    }
    return nil
}

    Triggers suportados¶

    Trigger Parâmetro Comportamento
    onlogon /SC ONLOGON Executa a cada logon de qualquer usuário
    daily /SC DAILY /ST 09:00 Executa diariamente às 09:00
    hourly /SC HOURLY Executa a cada hora

    Flags importantes¶

    • /F: Force -- sobrescreve tarefa existente com o mesmo nome
    • /RL HIGHEST: Run Level -- executa com o nível de privilégio mais alto disponível
    • /TN: Task Name -- nome da tarefa no Task Scheduler

    Naming¶

    Assim como nas Run keys, o nome da tarefa deve se mimetizar com tarefas legítimas. Exemplos:

    • \Microsoft\Windows\WindowsUpdate\UpdateCheck
    • \Microsoft\Office\OfficeBackgroundTaskHandlerRegistration
    • OneDrive Standalone Update Task

    Security Context¶

    Com /RL HIGHEST, a tarefa executa com o token mais elevado disponível:

    • Se o usuário é administrador: executa com token elevado (equivalente a "Run as Administrator")
    • Se configurada para SYSTEM: executa como NT AUTHORITY\SYSTEM

    Considerações Operacionais¶

    schtasks.exe vs COM API¶

    Aspecto schtasks.exe COM API (ITaskService)
    Processo filho Sim (schtasks.exe no tree) Não
    Complexidade Baixa Alta (COM boilerplate)
    Logging Event ID 4698 + Process Create Event ID 4698
    Compatibilidade Universal Requer COM initialization

    O Wolfgang atualmente usa schtasks.exe. A COM API seria mais furtiva por não spawnar um processo filho observável, mas adiciona complexidade significativa ao código.

    Cleanup¶

    Para remover a tarefa ao final da operação:

    schtasks.exe /Delete /TN "TaskName" /F

    Sobrevivência¶

    Scheduled tasks sobrevivem a:

    • Reboot
    • Logoff/logon
    • Password changes do usuário (se configurada para SYSTEM)
    • Atualizações do Windows

    Detecção¶

    IOC Descrição
    Event ID 4698 (Security log) Criação de scheduled task
    Event ID 106 (Task Scheduler log) Tarefa registrada
    Event ID 200/201 (Task Scheduler log) Execução de tarefa
    Sysmon Event ID 1 Processo schtasks.exe com argumentos /Create
    Arquivo XML em C:\Windows\System32\Tasks\ Definição da tarefa em disco

    Mitigações:

    • Monitorar criação de tarefas: Alertar sobre Event ID 4698 para tarefas não-whitelist
    • Restringir schtasks.exe: AppLocker rule para limitar quem pode criar tarefas
    • Auditar Tasks folder: Detectar arquivos XML novos ou modificados
