    Injeção de Processo¶

    Injeção de processo permite executar código no contexto de outro processo, evitando atribuição direta ao implant. Isso dificulta a análise forense e permite que payloads executem sob a identidade de processos legítimos.

    O Wolfgang implementa múltiplas técnicas de injeção, cada uma com tradeoffs diferentes de furtividade, complexidade e compatibilidade.

    Técnicas¶

    Process Hollowing¶

    Cria um processo legítimo suspense, desmapeia a imagem original e mapeia o payload no lugar. O processo resultante aparece como o binário original para ferramentas de monitoramento.

    • Vantagem: Processo aparece legítimo no Task Manager e ferramentas de análise
    • Desvantagem: Imagem em memória não corresponde ao disco (detecção por image integrity check)
    • MITRE: T1055.012

    EarlyBird APC¶

    Cria um processo suspense, aloca memória, escreve shellcode e enfileira um APC na thread principal. O APC executa antes do entry point do processo, dando ao shellcode a primeira oportunidade de execução.

    • Vantagem: Execução antes de qualquer código do processo alvo e hooks de EDR
    • Desvantagem: O shellcode não tem o contexto do PE original (imports, TLS)
    • MITRE: T1055.004

    Module Stomping¶

    Carrega uma DLL legítima no processo e sobrescreve sua seção .text com o payload. A memória resultante é disk-backed (apontando para a DLL legítima), evitando detecção por unbacked memory heuristics.

    • Vantagem: Memória aparece como pertencente a uma DLL legítima
    • Desvantagem: Seção .text modificada pode ser detectada por integrity check
    • MITRE: T1055.001

    Reflective Loading¶

    Implementa um PE loader completo em memória: parse de headers, mapeamento de seções, relocations, resolução de imports e configuração de permissões. O módulo é carregado sem usar a Windows Loader API (LoadLibrary).

    • Vantagem: Nenhuma chamada para LoadLibrary (sem eventos de module load)
    • Desvantagem: Memória sem backing em disco (unbacked, alta suspeita)
    • MITRE: T1620.001

    Comparação¶

    Técnica Alvo Stealth Complexidade Unbacked Memory
    Process Hollowing Processo novo Alta Alta Sim
    EarlyBird APC Processo novo Alta Média Sim
    Module Stomping Processo atual Muito Alta Média Nao
    Reflective Loading Processo atual Média Muito Alta Sim

    Interface Unificada¶

    Todas as técnicas são expostas via ProcessInjector, que seleciona a implementação baseado no argumento technique:

    type ProcessInjector struct {
    injectFunc      func(pid uint32, shellcode []byte) error       // CRT
    earlyBirdFunc   func(targetExe string, shellcode []byte) error // EarlyBird
    hollowingFunc   func(targetExe string, payload []byte) error   // Hollowing
}

func (p *ProcessInjector) Execute(ctx context.Context, args module.ModuleArgs) {
    switch technique {
    case "crt":       p.executeCRT(parsed, shellcode)
    case "earlybird": p.executeEarlyBird(parsed, shellcode)
    case "hollowing": p.executeHollowing(parsed, shellcode)
    }
}

    Cadeia de Evasão Recomendada¶

    Para máxima furtividade, as técnicas de injection devem ser combinadas com as técnicas de evasão:

    1. Sandbox Detection  → ambiente safe?
2. AMSI Bypass (HWBP) → desabilitar scanning
3. ETW Bypass (HWBP)  → silenciar telemetria
4. Direct Syscalls    → resolver SSNs
5. Injection          → executar payload
6. Sleep Obfuscation  → proteger entre beacons
