    Plugins¶

    O Wolfgang usa Hashicorp go-plugin para extensibilidade. Plugins rodam como processos separados e comunicam com o server via gRPC. Isso fornece isolamento de crash, verificacao de integridade e a possibilidade de escrever plugins em qualquer linguagem que suporte gRPC.

    Por que go-plugin?¶

    Alternativa Por que nao
    Shared libraries (.so/.dll) Crash no plugin derruba o server. Sem isolamento de memoria
    Go plugins (plugin stdlib) Linux-only, mesma versao de Go requerida, sem crash isolation
    Microservices HTTP Overhead de deploy e discovery. O plugin precisa gerenciar seu proprio lifecycle
    go-plugin (gRPC) Isolamento de processo, crash safety, AutoMTLS, language agnostic

    Beneficios concretos¶

    • Crash isolation: Se um plugin panics ou segfaults, o server continua operando. O Manager detecta a morte do processo e marca o plugin como unloaded
    • Language agnostic: O SDK define interfaces gRPC via protobuf. Um plugin pode ser escrito em Python, Rust, ou qualquer linguagem com suporte gRPC
    • AutoMTLS: go-plugin gera certificados TLS automaticamente para a comunicacao gRPC entre server e plugin. Zero configuracao
    • Versioning: Handshake protocol com magic cookie garante compatibilidade

    Tipos de Plugin¶

    // pkg/plugin/plugin.go
type PluginType string

const (
    ListenerPlugin PluginType = "listener" // fornece novos transport listeners
    ModulePlugin   PluginType = "module"   // fornece novos modulos de execucao
    FullPlugin     PluginType = "full"     // fornece ambos
)
    Tipo Interfaces Exemplo
    listener InfoProvider + Listener Plugin que implementa listener SMB
    module InfoProvider + ModuleProvider Plugin que adiciona modulo de AD enumeration
    full InfoProvider + Listener + ModuleProvider Plugin completo com listener + modulos

    Handshake¶

    O handshake garante que server e plugin estao na mesma versao do protocolo:

    // pkg/pluginsdk/grpc.go
var Handshake = plugin.HandshakeConfig{
    ProtocolVersion:  1,
    MagicCookieKey:   "WOLFGANG_PLUGIN",
    MagicCookieValue: "wolfgang-v1",
}

    Se o magic cookie nao bater, go-plugin rejeita o plugin imediatamente. Isso previne execucao acidental de binarios que nao sao plugins Wolfgang.

    Verificacao SHA-256¶

    Quando verify_hash: true no config, o Manager calcula o SHA-256 de cada binario e compara com a whitelist:

    // pkg/plugin/manager.go
func (m *Manager) loadPlugin(path string) error {
    hash, _ := hashFile(path)

    if m.verifyHash {
        if !m.whitelist[hash] {
            return fmt.Errorf("hash %s not in whitelist", hash)
        }
    }
    // ...
}

func hashFile(path string) (string, error) {
    f, _ := os.Open(path)
    h := sha256.New()
    io.Copy(h, f)
    return hex.EncodeToString(h.Sum(nil)), nil
}

    Comunicacao gRPC¶

    sequenceDiagram
    participant S as Wolfgang Server
    participant M as Plugin Manager
    participant P as Plugin Process

    Note over M,P: Startup
    M->>P: exec.Command(pluginPath)
    P->>P: plugin.Serve(config)
    Note over P: Inicia gRPC server
    com AutoMTLS

    P-->>M: stdout: plugin protocol info
    M->>P: gRPC Connect (mTLS)

    Note over M,P: Discovery
    M->>P: Dispense("info")
    P-->>M: InfoProvider proxy
    M->>P: GetInfo()
    P-->>M: {Name, Version, Type, ListenerTypes, ModuleNames}

    Note over M,P: Operacao (Listener)
    M->>P: Dispense("listener")
    P-->>M: Listener proxy
    M->>P: StartListener({id, address, options})
    P-->>M: ListenerStatus{running: true}

    Note over M,P: Operacao (Module)
    M->>P: Dispense("module")
    P-->>M: ModuleProvider proxy
    M->>P: ExecuteModule({module, agentID, args})
    P-->>M: ModuleResult{success, output}

    Note over M,P: Shutdown
    M->>P: client.Kill()
    P->>P: Process termination

    Tres servicos gRPC¶

    Cada plugin expoe ate tres servicos gRPC:

    1. PluginInfo — Metadata (obrigatorio para todos os plugins):

    service PluginInfo {
    rpc GetInfo(Empty) returns (PluginInfoResponse);
}

    2. ListenerPlugin — Gerenciamento de listeners:

    service ListenerPlugin {
    rpc StartListener(StartListenerRequest) returns (ListenerStatus);
    rpc StopListener(StopListenerRequest) returns (ListenerStatus);
}

    3. ModulePlugin — Execucao de modulos:

    service ModulePlugin {
    rpc ExecuteModule(ExecuteModuleRequest) returns (ModuleResult);
    rpc ListModules(Empty) returns (ModuleList);
}

    Plugin Map¶

    O go-plugin usa um mapa de "plugin names" para conectar servicos:

    // pkg/pluginsdk/grpc.go
var PluginMap = map[string]plugin.Plugin{
    "info":     &InfoGRPCPlugin{},
    "listener": &ListenerGRPCPlugin{},
    "module":   &ModuleGRPCPlugin{},
}

    SDK — Interfaces para Plugin Authors¶

    O pacote pkg/pluginsdk e a API publica para escrever plugins.

    Interfaces¶

    // pkg/pluginsdk/sdk.go

// Listener — implementar para plugins de transport
type Listener interface {
    Start(cfg ListenerConfig) (ListenerStatus, error)
    Stop(id string) error
}

type ListenerConfig struct {
    ID      string            `json:"id"`
    Address string            `json:"address"`
    Options map[string]string `json:"options,omitempty"`
}

// Module — implementar para plugins de modulo
type Module interface {
    Name() string
    Execute(args ModuleArgs) ModuleResult
}

// ModuleProvider — wraps multiplos modules
type ModuleProvider interface {
    Execute(module, agentID string, args map[string]string) (ModuleResult, error)
    List() []string
}

    Helper Functions para Servir Plugins¶

    O SDK fornece funcoes helper que encapsulam toda a configuracao do go-plugin:

    Plugin de Listener:

    // pkg/pluginsdk/serve.go
func ServeListener(name, version string, listener Listener) {
    plugin.Serve(&plugin.ServeConfig{
        HandshakeConfig: Handshake,
        Plugins: map[string]plugin.Plugin{
            "listener": &ListenerGRPCPlugin{Impl: listener},
            "info":     &InfoGRPCPlugin{Impl: info},
            "module":   &ModuleGRPCPlugin{}, // empty — nao implementa
        },
        GRPCServer: plugin.DefaultGRPCServer,
    })
}

    Plugin de Module:

    func ServeModule(name, version string, modules []Module) {
    // Agrupa modulos em ModuleProvider
    moduleMap := make(map[string]Module, len(modules))
    for _, m := range modules {
        moduleMap[m.Name()] = m
    }

    plugin.Serve(&plugin.ServeConfig{
        HandshakeConfig: Handshake,
        Plugins: map[string]plugin.Plugin{
            "module":   &ModuleGRPCPlugin{Impl: &moduleCollection{modules: moduleMap}},
            "info":     &InfoGRPCPlugin{Impl: info},
            "listener": &ListenerGRPCPlugin{}, // empty
        },
        GRPCServer: plugin.DefaultGRPCServer,
    })
}

    package main

import "github.com/user/wolfgang/pkg/pluginsdk"

type smbListener struct{}

func (s *smbListener) Start(cfg pluginsdk.ListenerConfig) (pluginsdk.ListenerStatus, error) {
    // Inicia listener SMB...
    return pluginsdk.ListenerStatus{ID: cfg.ID, Running: true, Address: cfg.Address}, nil
}

func (s *smbListener) Stop(id string) error {
    // Para listener...
    return nil
}

func main() {
    pluginsdk.ServeListener("smb-listener", "1.0.0", &smbListener{})
}

    Manager — Gerenciamento de Plugins¶

    O Manager gerencia o lifecycle completo:

    // pkg/plugin/manager.go
type Manager struct {
    dir        string           // diretorio de plugins
    verifyHash bool             // verificar SHA-256?
    whitelist  map[string]bool  // hashes permitidos
    audit      AuditFunc        // callback de auditoria
    plugins    map[string]*runningPlugin
}

    Operacoes¶

    Metodo Descricao
    Scan() Descobre e carrega todos os plugins do diretorio
    Reload() Para todos e re-scana (hot reload)
    StopAll() Para todos os plugins gracefully
    List() Retorna info de todos os plugins carregados
    GetListenerPlugin(type) Busca plugin de listener por tipo
    GetModulePlugin(name) Busca plugin de modulo por nome
    ListenerTypes() Lista todos os tipos de listener disponíveis via plugins
    ModuleNames() Lista todos os nomes de modulo disponíveis via plugins

    Panic Recovery¶

    O carregamento de plugins inclui panic recovery para evitar que um plugin malformado derrube o server:

    func (m *Manager) loadPlugin(path string) (retErr error) {
    defer func() {
        if r := recover(); r != nil {
            retErr = fmt.Errorf("panic during load: %v", r)
        }
    }()
    // ...
}

    Modelo de Seguranca¶

    Aspecto Protecao
    Supply chain SHA-256 hash verification antes do carregamento
    Runtime isolation Processo separado — crash nao afeta server
    Transport AutoMTLS (certificados efemeros gerados pelo go-plugin)
    Protocol Magic cookie handshake previne execucao de binarios arbitrarios
    Auditoria Eventos plugin.load e plugin.unload registrados
    Least privilege Plugin so acessa as interfaces gRPC expostas, nao a memoria do server
