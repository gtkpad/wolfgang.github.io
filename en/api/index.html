    Referencia da API¶

    API REST do teamserver Wolfgang para gerenciamento de agents, tarefas e operadores.

    Visao Geral¶

    A API do Wolfgang segue o padrao REST sobre HTTPS com autenticacao via JWT Bearer token. Todas as respostas sao em JSON e seguem a estrutura padrao api.Response.

    Base URL¶

    https://<teamserver>:<port>/api/v1

    Autenticacao¶

    Todas as rotas requerem um token JWT no header Authorization:

    Authorization: Bearer <jwt_token>

    O token e obtido via login no endpoint de autenticacao.

    Estrutura de Resposta¶

    Todas as respostas seguem o envelope padrao:

    {
  "success": true,
  "data": { ... },
  "error": ""
}

    Em caso de erro:

    {
  "success": false,
  "data": null,
  "error": "descricao do erro"
}

    Endpoints Principais¶

    Agents¶

    Metodo Path Descricao
    GET /agents Listar todos os agents registrados
    GET /agents/:id Detalhes de um agent
    DELETE /agents/:id Remover agent

    Tarefas¶

    Metodo Path Descricao
    POST /agents/:id/tasks Criar tarefa para um agent
    GET /agents/:id/tasks Listar tarefas de um agent
    GET /tasks/:id Detalhes de uma tarefa

    Operadores¶

    Metodo Path Descricao
    POST /auth/login Autenticar e obter JWT
    GET /operators Listar operadores

    Listeners¶

    Metodo Path Descricao
    GET /listeners Listar listeners ativos
    POST /listeners Criar listener
    DELETE /listeners/:id Remover listener

    Builds¶

    Metodo Path Descricao
    POST /generate Gerar implant customizado

    Documentacao Interativa¶

    A especificacao OpenAPI completa esta disponivel via Swagger UI:

    O arquivo swagger.json contem a especificacao Swagger 2.0 completa com todos os schemas, parametros e exemplos de resposta.

    Acesso Local¶

    Ao executar o teamserver localmente, a documentacao interativa esta disponivel em:

    https://localhost:<port>/swagger/index.html

    Exemplos de Uso¶

    Autenticacao¶

    curl -X POST https://teamserver:443/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username": "operator1", "password": "s3cret"}'

    Listar Agents¶

    curl -s https://teamserver:443/api/v1/agents \
  -H "Authorization: Bearer <token>" | jq

    Criar Tarefa¶

    curl -X POST https://teamserver:443/api/v1/agents/<agent_id>/tasks \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "module": "shell.exec",
    "args": "{\"command\": \"whoami\"}"
  }'

    Gerar Implant¶

    curl -X POST https://teamserver:443/api/v1/generate \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{
    "os": "windows",
    "arch": "amd64",
    "listener_id": "<listener_id>"
  }' -o implant.exe

    Seguranca¶

    • Todas as comunicacoes sao via HTTPS (TLS 1.2+)
    • Tokens JWT tem expiracao configuravel
    • Rate limiting e aplicado por IP
    • Logs de auditoria registram todas as operacoes
