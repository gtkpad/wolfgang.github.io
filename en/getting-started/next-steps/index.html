    Topicos Avancados por Area¶

    Transports e Comunicacao¶

    O Wolfgang suporta tres transportes, cada um com trade-offs diferentes:

    Transport Vantagem Quando usar
    HTTPS Mistura com trafego web, simples de configurar Default para a maioria dos cenarios
    DNS Atravessa firewalls restritivos, dificil de bloquear Redes com saida limitada
    WebSocket Conexao persistente, menor latencia Operacoes que precisam de respostas rapidas

    O implant suporta transport chain: se o transporte primario falhar, ele tenta automaticamente os fallbacks configurados. Veja Operator Guide: Listeners para configurar multiplos transportes.

    Evasion e Tradecraft¶

    Capacidades de evasao disponiveis no agent:

    Tecnica Comando Plataforma
    AMSI bypass evasion amsi [strategy] Windows
    ETW bypass evasion etw [strategy] Windows
    Ntdll unhooking evasion unhook Windows
    Process injection evasion inject <tech> <pid> <shellcode> Windows
    Shellcode execution evasion memexec <shellcode_file> Windows
    Module stomping evasion stomp <shellcode> [dll] Windows
    Syscall resolution evasion syscall Windows

    Muitas dessas tecnicas podem ser habilitadas automaticamente no build do implant via flags como --auto-amsi, --auto-etw, --auto-sandbox. Veja Tradecraft: Evasion para detalhes.

    Credential Access¶

    O Wolfgang inclui modulos de coleta de credenciais:

    Modulo Comando Descricao
    LSASS dump creds lsass [comsvcs\|minidump\|direct] Dump de processo LSASS (3 metodos)
    Kerberoasting creds kerberoast Extrai tickets Kerberos para cracking offline
    Registry hives creds hive [sam\|security\|system\|all] Export de hives SAM/SECURITY/SYSTEM
    Credential vaults creds vault [credman\|browser\|wifi\|all] Credential Manager, browsers, WiFi

    Detalhes em Module Catalog.

    Lateral Movement e Pivoting¶

    Funcionalidade Comando Descricao
    Remote execution rexec <host> <cmd> [--method wmi\|scm] Executar comando remoto via WMI ou SCM
    Network pivoting pivot <start\|stop\|list> [target:port] Pivotar pela rede via agent
    Port forwarding portfwd <add\|remove> <port> <addr:port> Redirecionar portas via agent
    SOCKS5 proxy socks <start\|stop> [port] Proxy SOCKS5 via agent

    Persistence¶

    Mecanismos de persistencia suportados:

    Metodo Comando Plataforma
    Registry Run key persist registry <exe_path> Windows
    Scheduled task persist schtask <exe_path> Windows
    Cron job persist cron <command> [schedule] Linux
    LaunchDaemon persist launchd <exe_path> macOS

    O comando cleanup remove todos os artefatos de persistencia instalados, e selfdestruct termina permanentemente o implant.

    Geracao Avancada de Implants¶

    Opcoes avancadas de build para cenarios especificos:

    Feature Descricao
    Domain fronting Esconder C2 atras de CDNs (CloudFront, Azure CDN)
    Malleable profiles Customizar headers HTTP, URIs, e timing
    Stager mode Binario minimo que baixa o implant completo em runtime
    Polymorphic builds Cada compilacao gera um binario com hash unico
    Dynamic modules Carregar modulos sob demanda ao inves de embutir todos
    Sleep encryption Criptografar heap durante periodos de sleep

    Veja Operator Guide: Generate para configuracao detalhada.

    Desenvolvimento de Modulos¶

    O sistema de modulos do Wolfgang usa a interface Module:

    type Module interface {
    Name() string
    Execute(ctx context.Context, args ModuleArgs) (ModuleResult, error)
    OS() []string
}

    Modulos sao registrados no registry com filtragem automatica por plataforma. Modulos dinamicos podem ser carregados em runtime sem recompilar o implant.

    Guia completo em Developer Guide: Adding Modules.

    API REST¶

    O teamserver expoe uma API REST completa documentada via Swagger:

    https://<server>:8443/swagger/index.html

    Endpoints principais:

    Endpoint Metodo Descricao
    /api/v1/agents GET Listar agents
    /api/v1/agents/{id}/tasks POST Criar task
    /api/v1/tasks/{id} GET Consultar task
    /api/v1/listeners GET/POST Gerenciar listeners
    /api/v1/operators GET/POST Gerenciar operadores
    /api/v1/generate POST Gerar implant
    /api/v1/exfil GET Listar sessoes de exfiltracao
    /api/v1/logs GET Audit logs

    Autenticacao via Bearer token JWT. Veja API Reference para documentacao completa.

    Plugin System¶

    O Wolfgang suporta plugins externos que estendem a funcionalidade do servidor:

    • Plugins ficam em ~/.wolfgang/plugins/
    • Verificacao de hash SHA256 habilitada por padrao
    • Lifecycle gerenciado pelo servidor (scan, start, stop)

    Detalhes em Developer Guide: Plugins.

    Operacoes Seguras¶

    Algumas boas praticas para uso em engagements reais:

    1. Jitter alto em producao: Use --jitter 30-50 para evitar padroes detectaveis de check-in
    2. Sleep adequado: --sleep 60000 (1 min) ou mais para operacoes de longa duracao
    3. Transport chain: Configure fallbacks para resiliencia
    4. Cleanup sempre: Execute cleanup antes de encerrar o engagement
    5. Logs de auditoria: Use logs para manter registro de todas as operacoes
    6. Operadores dedicados: Crie operadores individuais para cada membro da equipe
