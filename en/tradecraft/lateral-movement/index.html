    Movimento Lateral¶

    Lateral movement é o conjunto de técnicas que permitem ao operador expandir o acesso de uma máquina comprometida para outras na rede. É a transição de "um implant em uma máquina" para "presença em múltiplos segmentos da rede".

    O Wolfgang implementa duas categorias de lateral movement:

    1. Network Pivoting: Transformar o implant em um relay de rede para acessar segmentos internos sem conectividade direta com o operador
    2. Remote Execution: Executar comandos em máquinas remotas usando credenciais obtidas via Credential Access

    Técnicas Disponíveis¶

    Network Pivoting¶

    Técnica Descrição Plataforma MITRE
    Port Forwarding TCP relay local para alvo remoto Todas T1090
    SOCKS Proxy Proxy SOCKS5 para roteamento dinâmico Todas T1090.001
    Pivot Chains Encadeamento de pivots para redes segmentadas Todas T1090.003

    Remote Execution¶

    Técnica Descrição Plataforma MITRE
    Remote Execution SCM e WMI para execução em hosts remotos Windows T1021.002, T1047

    Network Pivoting vs Remote Execution¶

    Aspecto Network Pivoting Remote Execution
    Propósito Alcançar redes inaccessíveis Executar código em hosts remotos
    Credenciais Não requer Requer (NTLM hash, ticket, senha)
    Plataforma Cross-platform Windows-only (SCM/WMI)
    Footprint Tráfego de rede Serviço criado / processo remoto
    Implant no alvo Não automaticamente Pode deployar implant

    Uso combinado¶

    O cenário mais comum combina ambas categorias:

    1. SOCKS Proxy no implant A para alcançar a rede do segmento B
    2. Remote Execution (via SOCKS) para executar o implant na máquina B
    3. Pivot Chain do implant B para alcançar o segmento C

    Topologias Comuns¶

    graph TB
    subgraph "Estrela (Star)"
        S_OP[Operador] --> S_A[Implant A]
        S_A --> S_B[Implant B]
        S_A --> S_C[Implant C]
        S_A --> S_D[Implant D]
    end

    A topologia estrela é a mais simples: um implant central (pivot) conecta o operador a múltiplos alvos. Risco: perder o pivot central desconecta todos.

    Para topologias mais complexas (chain, tree, mesh), veja Pivot Chains.
