    Tradecraft¶

    Tradecraft no contexto de C2 refere-se ao conjunto de técnicas operacionais usadas para executar ações no alvo sem ser detectado. O Wolfgang implementa tradecraft em camadas, onde cada técnica pode ser usada isoladamente ou composta com outras para criar cadeias de evasão mais robustas.

    Categorias¶

    Evasão¶

    Técnicas para evitar detecção por EDRs, antivírus e ferramentas de monitoramento. Inclui bypass de AMSI/ETW, direct syscalls, sleep obfuscation e sandbox detection.

    Técnica Plataforma MITRE ATT&CK
    AMSI Bypass Windows T1562.001
    ETW Bypass Windows T1562.006
    Sandbox Detection All T1497.001
    Sleep Obfuscation Windows T1027.012
    Direct Syscalls Windows T1106
    Memory Encryption Windows T1027
    Hardware Breakpoint Bypass Windows T1562.001

    Injeção¶

    Técnicas para executar código no contexto de outros processos, evitando atribuição direta ao implant.

    Técnica Plataforma MITRE ATT&CK
    Process Hollowing Windows T1055.012
    EarlyBird APC Windows T1055.004
    Module Stomping Windows T1055.001
    Reflective Loading Windows T1620.001

    Persistência¶

    Mecanismos para sobreviver a reboots e manter acesso contínuo ao alvo.

    Credential Access¶

    Extração de credenciais da memória, registro e armazenamentos seguros do sistema.

    Lateral Movement¶

    Técnicas para mover-se entre hosts dentro da rede alvo.

    Arquitetura de Composição¶

    As técnicas de tradecraft no Wolfgang são implementadas como modules independentes que seguem a interface module.Module. Isso permite composição livre:

    ┌─────────────────────────────────────────┐
│           Cadeia de Evasão              │
│                                         │
│  Sandbox Detection                      │
│       ↓ (safe)                          │
│  AMSI Bypass (HWBP)                     │
│       ↓                                 │
│  ETW Bypass (HWBP)                      │
│       ↓                                 │
│  Direct Syscalls (resolve SSNs)         │
│       ↓                                 │
│  Process Injection (EarlyBird)          │
│       ↓                                 │
│  Sleep Obfuscation (loop)               │
└─────────────────────────────────────────┘

    Cada módulo pode ser ativado ou desativado independentemente via configuração do implant no momento da geração ou em runtime via comandos do operador.
