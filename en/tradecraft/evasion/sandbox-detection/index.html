    Sandbox Detection¶

    O que é¶

    Sandbox detection é o processo de identificar se o implant está executando em um ambiente de análise (sandbox, VM de detonação, debugger) antes de realizar ações maliciosas. Quando um sandbox é detectado, o implant pode sair silenciosamente, dormir indefinidamente ou continuar com comportamento benigno.

    O Wolfgang implementa um sistema de scoring ponderado com múltiplas heurísticas que avaliam hardware, timing, rede e artefatos de usuário para tomar uma decisão composite sobre a legitimidade do ambiente.

    Por que funciona¶

    Sandboxes e VMs de análise tipicamente têm características que as distinguem de workstations reais:

    • Recursos limitados: Poucas CPUs, pouca RAM, discos pequenos
    • Hardware virtual: MACs de adaptadores de rede com OUI de hypervisors
    • Ambiente estéril: Poucos programas instalados, diretórios de usuário vazios
    • Timing artificial: Sleep acceleration para acelerar análise
    • Sem atividade humana: Nenhum input recente de mouse/teclado

    Cada uma dessas características isoladamente pode ter false positives, mas a combinação ponderada produz uma decisão robusta. O sistema de scoring do Wolfgang exige que múltiplos indicadores sejam positivos antes de classificar o ambiente como sandbox.

    Implementação no Wolfgang¶

    Configuração¶

    type SandboxConfig struct {
    MinCPU           int           // mínimo de CPUs (default: 2)
    MinRAMBytes      uint64        // mínimo de RAM (default: 2GB)
    MinUptimeSeconds float64       // mínimo de uptime (default: 300s)
    MinDiskBytes     uint64        // mínimo de disco (default: 60GB)
    MinPrograms      int           // mínimo de programas instalados (default: 10)
    ScoreThreshold   int           // limiar de detecção (default: 40)
    OnDetection      DetectionAction // ação ao detectar (Exit/Sleep/Proceed)
}

    Os defaults são conservadores -- um desktop típico ultrapassa todos os limiares facilmente, enquanto sandboxes frequentemente falham em 2-3 checks.

    Sistema de Scoring¶

    Cada check tem um peso que reflete seu poder discriminatório:

    func (d *SandboxDetector) RunScoredChecks() (bool, SandboxScore) {
    scored := []ScoredCheck{
        {d.checkCPU(), 20},
        {d.checkRAM(), 20},
        {d.checkUptime(), 15},
        {d.checkMAC(), 30},              // alto: VM MAC é forte indicador
        {d.checkUserActivity(), 15},
        {d.checkDisk(), 15},
        {d.checkInstalledPrograms(), 10},
        {d.checkSleepAcceleration(), 25}, // alto: muito específico
        {d.checkGPU(), 10},
        {d.checkUserArtifacts(), 10},
    }
    // Platform-specific checks adicionados via platformScoredChecks()

    total := 0
    for _, s := range scored {
        if !s.Check.Safe {
            total += s.Weight
        }
    }
    return total < threshold, SandboxScore{...}
}

    Checks Implementados¶

    Hardware¶

    func (d *SandboxDetector) checkCPU() CheckResult {
    count := d.sysInfo.CPUCount()
    if count < d.config.MinCPU {
        return CheckResult{Name: "cpu", Safe: false,
            Reason: fmt.Sprintf("low CPU count: %d", count)}
    }
    return CheckResult{Name: "cpu", Safe: true}
}
    • CPU (peso 20): Verifica contagem de processadores logicos
    • RAM (peso 20): Verifica memória total do sistema
    • Disk (peso 15): Verifica tamanho do disco principal
    • GPU (peso 10): Verifica presença de GPU dedicada

    Rede¶

    • MAC (peso 30): Compara OUI prefixes com hypervisors conhecidos
    • Network Adapters (peso 5, Windows): Sandboxes tipicamente têm <= 2 adapters

    Timing¶

    func (d *SandboxDetector) checkSleepAcceleration() CheckResult {
    start := time.Now()
    time.Sleep(1000 * time.Millisecond)
    elapsed := time.Since(start)
    if elapsed < 900*time.Millisecond {
        return CheckResult{Name: "sleep_timing", Safe: false,
            Reason: fmt.Sprintf("sleep accelerated: got %dms", elapsed.Milliseconds())}
    }
    return CheckResult{Name: "sleep_timing", Safe: true}
}
    • Sleep Acceleration (peso 25): Sandboxes aceleram Sleep() para concluir análise
    • Uptime (peso 15): Sandboxes são reiniciadas frequentemente (uptime < 5min)

    Artefatos de Usuário¶

    func (d *SandboxDetector) checkUserArtifacts() CheckResult {
    home, _ := os.UserHomeDir()
    dirs := []string{
        filepath.Join(home, "Desktop"),
        filepath.Join(home, "Documents"),
        filepath.Join(home, "Downloads"),
    }
    totalFiles := 0
    for _, dir := range dirs {
        entries, err := os.ReadDir(dir)
        if err == nil {
            totalFiles += len(entries)
        }
    }
    if totalFiles < 5 {
        return CheckResult{Name: "user_artifacts", Safe: false, ...}
    }
    return CheckResult{Name: "user_artifacts", Safe: true}
}
    • User Activity (peso 15): Verifica input recente de mouse/teclado
    • User Artifacts (peso 10): Conta arquivos em Desktop/Documents/Downloads
    • Installed Programs (peso 10): Ambientes reais têm 10+ programas

    Ação pós-detecção¶

    func (d *SandboxDetector) HandleDetection() bool {
    switch d.config.OnDetection {
    case ActionSleep:
        for { time.Sleep(24 * time.Hour) } // dorme indefinidamente
    case ActionProceed:
        return true  // ignora e continua
    default: // ActionExit
        return false // sair do processo
    }
}

    Cenários de Detecção¶

    Sandbox típico (Windows Defender sandbox)¶

    • CPU: 1 core (+20) / RAM: 1GB (+20) / MAC: Hyper-V (+30) / No user files (+10)
    • Score: 80 -- detectado (threshold: 40)

    VM de análise (Any.Run, Hybrid Analysis)¶

    • CPU: 2 cores / RAM: 4GB / MAC: KVM (+30) / Sleep accel (+25) / No activity (+15)
    • Score: 70 -- detectado

    Workstation real¶

    • CPU: 8 cores / RAM: 16GB / MAC: Intel / Uptime: 3 dias / 200+ files
    • Score: 0 -- safe

    False positive: servidor em cloud¶

    • CPU: 4 cores / RAM: 8GB / MAC: Hyper-V (+30) / Sem GUI activity (+15)
    • Score: 45 -- detectado (pode precisar ajustar threshold)

    Detecção¶

    IOC Descrição
    Queries de hardware incomuns WMI queries para CPU, RAM, GPU em sequência
    Enumeração de interfaces de rede Iteração sobre todos os adapters
    Leitura de diretórios de usuário Acesso a Desktop/Documents/Downloads
    Sleep timing measurement Sleep(1000ms) seguido de check de timestamp
    GetTickCount / QueryPerformanceCounter Chamadas de timing repetidas

    Contramedidas¶

    Sandboxes modernos podem:

    • Reportar hardware realista (4+ CPUs, 8GB+ RAM)
    • Usar MACs não associados a hypervisors
    • Pré-popular diretórios de usuário com arquivos falsos
    • Não acelerar Sleep (análise mais lenta, mas mais furtiva)

    O sistema de scoring do Wolfgang é configurável justamente para adaptar-se: operadores podem ajustar ScoreThreshold e pesos baseado no ambiente alvo.
