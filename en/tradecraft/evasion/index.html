    Evasão¶

    Técnicas de evasão permitem ao implant operar sem ser detectado por soluções de segurança como EDRs (Endpoint Detection and Response), antivírus e ferramentas de monitoramento do sistema operacional.

    O Wolfgang implementa evasão em múltiplas camadas, atuando em diferentes pontos da pilha de segurança do Windows:

    Camadas de Evasão¶

    ┌──────────────────────────────────────────────┐
│              Camada de Aplicação              │
│  AMSI Bypass ─── desabilita scanning inline  │
├──────────────────────────────────────────────┤
│             Camada de Telemetria             │
│  ETW Bypass ─── silencia event tracing       │
├──────────────────────────────────────────────┤
│               Camada de API                  │
│  Direct Syscalls ─── bypassa hooks em ntdll  │
├──────────────────────────────────────────────┤
│              Camada de Memória               │
│  Sleep Encryption ─── esconde payload idle   │
│  Memory Encryption ─── protege dados cached  │
├──────────────────────────────────────────────┤
│             Camada de Ambiente               │
│  Sandbox Detection ─── evita detonação       │
└──────────────────────────────────────────────┘

    Técnicas¶

    Bypass de Segurança¶

    • AMSI Bypass - Desabilita o Antimalware Scan Interface via memory patching ou hardware breakpoints. Impede que o CLR envie scripts e assemblies para o antivírus antes da execução.

    • ETW Bypass - Neutraliza Event Tracing for Windows, removendo a capacidade de EDRs e do Defender de receber eventos de telemetria do processo.

    • Hardware Breakpoint Bypass - Alternativa ao memory patching que usa debug registers (DR0-DR3) para interceptar chamadas sem modificar memória. Indetectável por integrity checks.

    Evasão de API¶

    • Direct Syscalls - Resolve syscall numbers (SSNs) diretamente do ntdll.dll em disco, bypassando hooks de userland instalados por EDRs. Implementa Hell's Gate, Halo's Gate e Tartarus' Gate para resolução resiliente.

    Evasão de Memória¶

    • Sleep Obfuscation - Criptografa regiões de memória do implant durante períodos de sleep, impedindo que memory scanners detectem o payload enquanto está inativo.

    • Memory Encryption - Criptografia AES-256-CTR para dados em memória, incluindo o module cache e buffers registrados para sleep encryption.

    Evasão de Ambiente¶

    • Sandbox Detection - Sistema de scoring ponderado que analisa hardware, timing, artefatos de usuário e indicadores de virtualização para determinar se o implant está executando em um ambiente de análise.

    Estratégia de Composição¶

    O Wolfgang permite selecionar a estratégia de bypass via argumento strategy:

    Valor Comportamento
    auto Tenta HWBP primeiro, faz fallback para memory patching
    hwbp Usa apenas hardware breakpoints
    patch Usa apenas memory patching

    A estratégia auto é o padrão recomendado: hardware breakpoints não modificam memória (evitando integrity checks), mas têm limite de 4 registros (DR0-DR3). Se todos estiverem em uso, o fallback para patching garante que o bypass ainda funcione.
