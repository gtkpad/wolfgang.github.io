    Redirectors¶

    Redirectors sao proxies reversos posicionados entre o implant e o teamserver. Eles protegem o IP real do teamserver, filtram trafego malicioso e servem um site decoy para analistas.

    Nginx como redirector¶

    A configuracao completa do redirector Nginx inclui multiplas camadas de hardening.

    Configuracao completa¶

    # Rate limiting zones
limit_req_zone $binary_remote_addr zone=c2limit:10m rate=30r/m;
limit_req_zone $binary_remote_addr zone=general:10m rate=10r/s;
limit_conn_zone $binary_remote_addr zone=connlimit:10m;

# User-Agent blocklist
map $http_user_agent $blocked_agent {
    default 0;
    ~*curl           1;
    ~*wget           1;
    ~*python-requests 1;
    ~*python-urllib   1;
    ~*libwww-perl     1;
    ~*Go-http-client  1;
    ~*nikto           1;
    ~*nmap            1;
    ~*masscan         1;
    ~*zgrab           1;
    ~*sqlmap          1;
    ~*gobuster        1;
    ~*feroxbuster     1;
    ~*ffuf            1;
    ~*nuclei          1;
    ~*httpx           1;
    ~*censys          1;
    ~*shodan          1;
    ""                1;  # Bloquear user-agent vazio
}

upstream teamserver {
    server 10.0.0.5:8443;
    keepalive 4;
}

server {
    listen 443 ssl;
    server_name redirector.example.com;

    ssl_certificate /etc/letsencrypt/live/redirector.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/redirector.example.com/privkey.pem;

    # TLS hardening
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
    ssl_session_tickets off;

    # Request limits
    client_max_body_size 1m;
    client_body_buffer_size 16k;
    client_header_buffer_size 1k;
    large_client_header_buffers 4 8k;

    # Timeouts
    client_body_timeout 10s;
    client_header_timeout 10s;
    send_timeout 10s;
    keepalive_timeout 15s;
    keepalive_requests 50;

    # Connection limits
    limit_conn connlimit 10;

    # Security headers
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "no-referrer" always;
    add_header Content-Security-Policy "default-src 'self'" always;
    add_header Permissions-Policy "camera=(), microphone=(), geolocation=()" always;

    server_tokens off;

    # Block analysis IPs
    deny 0.0.0.0/8;

    # Block by user-agent
    if ($blocked_agent) {
        return 444;
    }

    # Block common scanner paths
    location ~* ^/(\.env|\.git|\.svn|\.htaccess|wp-admin|wp-login|wp-content|
                   administrator|phpmyadmin|server-status|server-info|
                   cgi-bin|xmlrpc\.php|actuator|swagger|graphql) {
        access_log off;
        return 444;
    }

    # C2 profile routes
    location ~ ^/v1\.0/(me/chats|me/joinedTeams|communications) {
        limit_req zone=c2limit burst=5 nodelay;

        # Validar TODOS os headers esperados do profile
        if ($http_user_agent !~* "Teams") {
            return 200 '{"error":"not_found"}';
        }
        if ($http_content_type != "application/json") {
            return 200 '{"error":"not_found"}';
        }
        if ($http_accept != "application/json") {
            return 200 '{"error":"not_found"}';
        }

        proxy_pass https://teamserver;
        proxy_ssl_verify off;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        proxy_connect_timeout 5s;
        proxy_send_timeout 10s;
        proxy_read_timeout 30s;

        proxy_hide_header X-Powered-By;
        proxy_hide_header Server;
    }

    # Decoy site
    location / {
        limit_req zone=general burst=20 nodelay;
        root /var/www/decoy;
        index index.html;
    }
}

    Template parametrizado¶

    O Wolfgang inclui um template Nginx em configs/redirectors/nginx.conf.template com variaveis:

    Variavel Descricao
    {{.BackendURL}} Endereco do teamserver (ex: 10.0.0.5:8443)
    {{.ListenPort}} Porta HTTPS (ex: 443)
    {{.ServerName}} Hostname publico
    {{.CertPath}} Caminho do certificado
    {{.KeyPath}} Caminho da chave privada
    {{.DecoyRoot}} Document root do site decoy

    Hardening checklist¶

    Os seguintes itens sao obrigatorios para uso operacional:

    • [x] Rate limiting -- prevenir scanning e brute-force
    • [x] User-Agent blocklist -- bloquear ferramentas conhecidas
    • [x] Header validation -- validar TODOS os headers do profile, nao apenas User-Agent
    • [x] Certificado valido -- usar Let's Encrypt, NUNCA self-signed
    • [x] Scanner path blocking -- rejeitar paths comuns de enumeration
    • [x] Connection limits -- max 10 conexoes por IP
    • [ ] GeoIP filtering -- restringir por pais (requer ngx_http_geoip2_module)
    • [ ] JA3 filtering -- bloquear fingerprints TLS de ferramentas de analise

    GeoIP (opcional)¶

    Requer ngx_http_geoip2_module e banco MaxMind:

    geoip2 /usr/share/GeoIP/GeoLite2-Country.mmdb {
    $geoip2_country_code country iso_code;
}

map $geoip2_country_code $allowed_country {
    BR 1;
    US 1;
    default 0;
}

# No server block:
if ($allowed_country = 0) {
    return 200 '<!DOCTYPE html><html><body>OK</body></html>';
}

    Caddy como redirector¶

    Caddy oferece configuracao mais simples com TLS automatico via ACME:

    redirector.example.com {
    # Block analysis IPs
    @blocked remote_ip 0.0.0.0/8

    handle @blocked {
        respond 403
    }

    # C2 traffic - match profile paths + headers
    @c2traffic {
        path /v1.0/me/chats* /v1.0/me/joinedTeams* /beta/me/presence*
        header User-Agent *Teams*
    }

    handle @c2traffic {
        reverse_proxy 10.0.0.5:8443 {
            transport http {
                tls_insecure_skip_verify
            }
        }
    }

    # Decoy for everything else
    handle {
        root * /var/www/decoy
        file_server
    }
}

    Vantagens do Caddy¶

    • TLS automatico (Let's Encrypt integrado)
    • Configuracao declarativa mais simples
    • Reload sem downtime

    Limitacoes do Caddy¶

    • Menos opcoes de filtering granular que Nginx
    • Sem GeoIP nativo (requer plugin)
    • Sem JA3 filtering

    Domain fronting via CDN¶

    Domain fronting utiliza CDNs para ocultar o destino real do trafego C2.

    Como funciona¶

    1. Implant conecta no IP do CDN (ex: cdn.cloudfront.net)
    2. TLS SNI mostra o dominio do CDN
    3. Header HTTP Host aponta para o dominio real (c2.example.com)
    4. CDN roteia para o backend baseado no Host header

    Configuracao¶

    Na geracao do implant:

    Callback URL: https://d123456.cloudfront.net
Fronting Domain: d123456.cloudfront.net
Actual Host: c2.example.com

    No CDN (CloudFront, Azure CDN, etc.), configure o backend para apontar ao redirector ou teamserver.

    Consideracoes¶

    • Nem todos os CDNs permitem domain fronting (CloudFront restringiu em 2018)
    • Azure CDN e alternativas regionais podem funcionar
    • Teste sempre antes do engagement
    • O trafego aparece como HTTPS normal para o CDN

    Boas praticas¶

    Certificados¶

    • Sempre usar Let's Encrypt ou CA publica
    • Certificados self-signed sao flag imediato para analistas
    • Configurar renovacao automatica (certbot renew)

    Site decoy¶

    O site decoy deve ser:

    • Categorizado (use categorias que nao sejam bloqueadas pelo proxy corporativo)
    • Com conteudo realista (clone de site legitimo ou template)
    • Atualizado periodicamente
    • Com robots.txt, favicon, etc.

    Separacao de infra¶

    Componente Rede Acesso
    Redirector Internet publica Implants + analistas
    Teamserver Rede interna/VPN Apenas operadores
    Storage/Backup Rede isolada Apenas teamserver

    Rotacao¶

    • Use multiplos redirectors com DNS round-robin ou GeoDNS
    • Prepare redirectors de backup para rotacao rapida
    • Se um redirector for queimado, rotacione para o proximo
    • Configure fallback URLs no implant para resiliencia
